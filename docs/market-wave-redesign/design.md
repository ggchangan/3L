# 结构感知的波峰波谷统一判定框架 — 设计文档

> 版本：v2.0 | 日期：2026-06-02 | 状态：✅ 已实现并合入master

---

## 〇、问题背景

### 现状

当前系统中波峰波谷判定**散落在三层**，各有一套逻辑，全是BIAS20滞后型：

| 层级 | 函数 | 位置 | 算法 |
|:----|:-----|:-----|:----|
| **大盘**（中证全指） | `judge_peak_valley()` | `review_compute_service.py` | BIAS20四维打分 → 5档 |
| **行业板块**（90个） | 无独立峰谷判定，只有20日涨幅排名 | — | 仅涨幅排名，无峰谷 |
| **概念板块**（375个） | `judge_concept_wave()` | `concept_wave_service.py` | BIAS20 V5评分 → 5阶段 |

### 三个问题

1. **BIAS20太滞后** — 20日均价变化极慢，信号延迟5-10天。概念板块用的SMA20也是同样问题
2. **不分结构** — 所有结构用同一套阈值。但《量价原理》6.2节明确：上升趋势中的波谷（缩量回踩）和下降趋势中的波谷（恐慌抛售）量价语言**完全不同**
3. **逻辑重复** — 大盘和概念各写了一套BIAS20评分，行业板块反而没有峰谷判断。应该统一

### 设计目标

**一个核心函数，三个层都调它：**

```
                 judge_structure_wave(klines, structure?)
                       /          |          \
                      ▼           ▼           ▼
                大盘(中证全指)  行业板块(90)  概念板块(375)
                      |           |           |
                      ▼           ▼           ▼
                仓位控制      主线机会分类   波谷追踪
```

## 一、设计框架：结构必选 → 峰谷判定

```
输入：日K线数据（中证全指/行业板块/概念板块）
        │
        ▼
第一步：判定当前结构（复用 get_structure()，也可外部传入）
        │
        ├── 上升趋势 ────→ 用"上升趋势峰谷规则"
        ├── 下降趋势 ────→ 用"下降趋势峰谷规则"
        └── 区间震荡 ────→ 用"区间震荡峰谷规则"
                          │
                          ▼
第二步：按结构规则计算 pk_score / vl_score
                          │
                          ▼
第三步：输出5档（偏波峰/波中偏上/波中/波中偏下/偏波谷）+仓位建议
```

### 1.2 适用范围

| 层 | 输入数据 | 输出用途 | 当前替换目标 |
|:--|:--------|:--------|:-----------|
| 中证全指（大盘） | 200天日K线 | 大盘仓位建议 | `judge_peak_valley()` |
| 行业板块（90个） | 板块日K线 | 主线机会分类（波谷/波峰） | —（新增） |
| 概念板块（375个） | 板块日K线 | 波谷追踪页面 | `judge_concept_wave()` |
```

## 二、结构判定规则

### 2.1 结构输入

复用现有 `backend/core/ema_utils.py` → `get_structure(closes)`，返回：
- `'上升趋势'` / `'下降趋势'` / `'区间震荡'`

当前结构判定算法（EMA10极值位置法）：
```python
e10 = ema_list(closes, 10)[-15:]
max_pos = max(range(n), key=lambda i: e10[i])
min_pos = min(range(n), key=lambda i: e10[i])
if max_pos >= lq and min_pos <= fq:  结构 = '上升趋势'
elif min_pos >= lq and max_pos <= fq: 结构 = '下降趋势'
else: 结构 = '区间震荡'
```

### 2.2 结构对峰谷判定影响

| 结构 | 波谷应该什么样子 | 波峰应该什么样子 |
|:----|:----------------|:----------------|
| **上升趋势** | 缩量回踩EMA5/10，价波收窄，供应枯竭 | 连续大阳加速+天量，需求透支 |
| **下降趋势** | 恐慌抛售：放量大跌但价不下探+收下影 | 缩量反弹无法持续，成交量跟不上 |
| **区间震荡** | 区间底部+缩量企稳+下影线 | 区间顶部+放量滞涨+上影线/墓碑线 |

## 三、量价行为判定规则（按结构）

### 3.1 通用工具（复用已有）

```python
# 从 signal_detector/base.py 或 ema_utils.py
ema5 = ema_list(closes, 5)         # 5日EMA
ema10 = ema_list(closes, 10)       # 10日EMA
bias5 = (close - ema5) / ema5 * 100  # 乖离率5（替代BIAS20）
vol_ratio = cur_vol / vma5         # 量比（当日量/5日均量）
body_pct = abs(close - open) / open * 100  # 实体百分比
range_pct = (high - low) / open * 100      # 振幅
us_pct = (high - max(open, close)) / open * 100  # 上影线
ls_pct = (min(open, close) - low) / open * 100   # 下影线
```

### 3.2 上升趋势 — 峰谷规则

#### 波谷（缩量回踩型）：原文6.2.1"上升趋势中的波谷通常是第一种情况（缩量回踩）"

| # | 条件 | 权重 | 原文依据 |
|:-:|:----|:---:|:---------|
| ① | 结构=上升趋势 | 否决项 | "上升趋势中的波谷" |
| ② | 回踩：距最近EMA5低点幅度 -1%~-8% | 25% | "回踩"的定义 |
| ③ | 缩量：vol_ratio < 0.7（量能萎缩）| 30% | "成交量大幅萎缩" |
| ④ | 价波收窄：range_pct < 前5日均值 | 15% | "波动明显收窄" |
| ⑤ | 有支撑：close >= ema10 * 0.97 | 15% | "不破坏原有趋势" |
| ⑥ | bias5 < 2%（不处于加速区） | 15% | 排除加速段误判 |

得分 ≥ 60 → 波谷确认（vl_score +1）

#### 波峰（加速高潮型）：原文6.2.3"上升趋势中的波峰是第一种情况（加速或高潮）"

| # | 条件 | 权重 | 原文依据 |
|:-:|:----|:---:|:---------|
| ① | 结构=上升趋势 | 否决项 | "上升趋势中的波峰" |
| ② | 连续加速：近5日有≥3日涨幅>2% | 25% | "加速高潮" |
| ③ | 天量：vol_ratio > 1.8（量异常放大）| 25% | "最后的需求被透支" |
| ④ | 放量滞涨：vol_ratio > 1.3 **且** body_pct < 0.8% | 20% | 量放不上去=买力耗尽 |
| ⑤ | bias5 > 5%（明显偏离均线）| 15% | "加速"的量化近似 |
| ⑥ | 上影线：us_pct > 1.5% | 15% | 供应涌现信号 |

得分 ≥ 60 → 波峰确认（pk_score +1）

### 3.3 下降趋势 — 峰谷规则

#### 波谷（恐慌抛售型）：原文6.2.1"下降趋势中的波谷，出现恐慌性抛售后见底的概率更大"

| # | 条件 | 权重 | 原文依据 |
|:-:|:----|:---:|:---------|
| ① | 结构=下降趋势 | 否决项 | "下降趋势中的波谷" |
| ② | 大跌：当日跌幅 > -3% | 20% | "恐慌性抛售" |
| ③ | 放量：vol_ratio > 1.5 | 25% | "放量的" |
| ④ | 不破位：close 不低于前日最低价大幅下探（下影线长） | 25% | "股价不再大幅下跌" |
| ⑤ | 下影线：ls_pct > body_pct * 1.5 | 20% | 需求开始接盘 |
| ⑥ | 近3日连跌+今日收阳（或收在相对高位） | 10% | "收在相对高位" |

得分 ≥ 65 → 波谷确认（vl_score +1）

> **注意**：下降趋势中的缩量不是波谷信号。原文说"下降趋势中缩量并不能简单地判断为供应衰竭，因为下降趋势成交量会持续萎缩"。

#### 波峰（缩量反弹型）：原文6.2.3"下降趋势中的波峰，往往是成交量跟不上"

| # | 条件 | 权重 | 原文依据 |
|:-:|:----|:---:|:---------|
| ① | 结构=下降趋势 | 否决项 | "下降趋势中的波峰" |
| ② | 反弹后缩量：连续涨2天以上但量比 < 1.0 | 30% | "成交量跟不上" |
| ③ | 无法突破EMA5/10：close < ema10 | 25% | 反弹无力 |
| ④ | bias5从负转正又回落：前日bias5>0、今日bias5<0 | 20% | 反弹结束 |
| ⑤ | 阴线覆盖阳线：今日阴线实体覆盖前日阳线 | 25% | 供应重新占优 |

得分 ≥ 60 → 波峰确认（pk_score +1）

### 3.4 区间震荡 — 峰谷规则

#### 波谷（区间底+缩量企稳）

| # | 条件 |
|:-:|:----|
| ① | 结构=区间震荡 |
| ② | 价格在近期低点附近（距20日最低点<3%）|
| ③ | vol_ratio < 0.7 **或** 连续3日缩量 |
| ④ | 下影线 > 实体 **或** 收阳企稳 |
| ⑤ | bias5 从负转正 |

得分 ≥ 60 → 波谷确认

#### 波峰（区间顶+放量滞涨）

| # | 条件 |
|:-:|:----|
| ① | 结构=区间震荡 |
| ② | 价格在近期高点附近（距20日最高点<3%）|
| ③ | vol_ratio > 1.3 **且** body_pct < 1% (放量滞涨) |
| ④ | 上影线 > 1.5% **或** 墓碑线 |
| ⑤ | bias5 从正转负 |

得分 ≥ 60 → 波峰确认

## 四、总分计算与5档输出

### pk_score / vl_score

```python
pk_score = 0
vl_score = 0

# 按当前结构跑对应的规则
for each_peak_rule in rules[structure]['peak']:
    if rule.passed:
        pk_score += rule.weight_scaled

for each_valley_rule in rules[structure]['valley']:
    if rule.passed:
        vl_score += rule.weight_scaled

# 高分强制覆盖（防止假信号）
if bias5 > 10:    # 极端偏离 → 一定是偏波峰区域
    pk_score = max(pk_score, 3)
if bias5 < -10:   # 极端偏离 → 一定是偏波谷区域
    vl_score = max(vl_score, 3)
```

### 5档判定（保留现有仓位建议）

```python
if pk_score >= 3:
    position = '偏波峰', pct = '五成'
elif pk_score >= 2:
    position = '波中偏上', pct = '六至七成'
elif vl_score >= 3:
    position = '偏波谷', pct = '五至八成'
elif vl_score >= 2:
    position = '波中偏下', pct = '五至七成'
else:
    position = '波中', pct = '七至八成'
```

**阈值从原来的4分降为3分（结构分层后每类规则更精确、触发更少，3分即可确认）**

## 五、新方案 vs 现有概念波谷系统（V5）对比

### 5.1 现有 `judge_concept_wave()` 分析

**文件位置：** `server/backend/services/concept_wave_service.py` `judge_concept_wave()`

**算法：** 5维度加权打分 → 5阶段输出（波谷/波峰/上涨/下跌/波中）

| 维度 | 具体条件 | 权重 |
|:----|:--------|:----:|
| BIAS20位置 | bias20<-5% → +1分，<-8% →再+1分 | 2/5 |
| 量能萎缩 | 量比<0.7 → +1分，<0.5 →再+1分 | 2/5 |
| EMA10斜率 | 从负转平→+1分，转正→+1分 | 2/5 |
| 2σ偏差 | z<-1.5→+1分，z<-2.0→再+1分 | 2/5 |

**阶段判定阈值：** vl_score≥3 + 缩量 → 波谷；pk_score≥3 + 放量下跌 → 波峰

**回测结果（365天，375概念，80,459个测试点）：**
- 波谷（vl_score≥3+缩量）：**77.3%** 10日胜率，+2.00%均收益
- 波峰（pk_score≥3+放量下跌）：**61.3%** 10日跌率，-0.55%均收益

### 5.2 V5的优点

1. **多维度** — BIAS+量能+斜率+标准差，覆盖面广
2. **有回测数据** — 77.3%/61.3%胜率，数据驱动决策
3. **EMA10斜率检测** — "从负转平/转正 = 下跌趋缓" 逻辑与原文一致
4. **切入窗口概念** — vl_score≥3+BIAS20在-5%~-8%+缩量，实用性强

### 5.3 V5的问题

#### 核心问题：不分结构

375个概念板块用**同一套打分规则**。但原文6.2节明确：

| 结构 | 波谷应该是什么 |
|:----|:--------------|
| **上升趋势** | 缩量回踩EMA10（原文6.2.1↑）|
| **下降趋势** | 恐慌抛售放量不跌（原文6.2.1↓）|
| **区间震荡** | 区间底部缩量企稳 |

V5在上升趋势中看见"缩量+BIAS20<-5%"就判波谷——但强上升趋势中缩量回踩是**正常调整**，不是波谷，BIAS20可能根本不会<-5%。

V5在下降趋势中看见持续缩量也判波谷——但原文说**下降趋势中的缩量不能简单判断为供应衰竭**，因为持续阴跌本来就一直缩量。

#### 次要问题

| 问题 | 说明 |
|:----|:-----|
| **SMA20太滞后** | 用的是简单移动平均（SMA），比EMA还慢。SMA给20天前和昨天同等权重 |
| **条件冗余** | BIAS20<-5%（条件1）和z_score<-1.5（条件4）都在测"价格偏离均值多远"，实际只有1.5个独立维度 |
| **波峰判定弱** | pk_score条件有但输出里几乎不用，页面主要展示波谷/上涨/下跌/波中，波峰阶段基本不出现 |
| **行业板块缺失** | 375个概念有峰谷判断，90个行业板块反而没有 |

### 5.4 77.3%胜率的解读

80,459个测试点覆盖375个概念×241天。大部分板块在**大部分时间处于上涨或波中**。在这些阶段，价格自然有均值回归倾向——在低点判波谷后，回到均值的概率本来就高。

**这不是说信号不准，而是说胜率有"结构红利"**：V5在上涨趋势中抓回调底部的能力被高估了，在下降趋势中抓真底的能力被低估了。

新方案通过**先分结构再判定**，理论上能：
- 上升趋势中：更精准识别缩量回踩位置（而不是超跌反弹）
- 下降趋势中：识别恐慌抛售真底（而不是持续阴跌假底）
- 两种结构分开算，不被均值回归"结构红利"误导

### 5.5 新方案是否更科学

| 对比维度 | V5概念波谷 | 新设计 |
|:--------|:---------|:------|
| 结构感知 | ❌ 统一规则 | ✅ 上升/下降/震荡各一套 |
| 时效性 | ❌ SMA20滞后 | ✅ BIAS5 |
| 量价语言 | ⚠️ 只有缩量/放量 | ✅ 实体+影线+振幅+缩量+放量 |
| 与6.2原文匹配 | ❌ 统计偏离度思维 | ✅ 供应衰竭/需求透支思维 |
| 统一性 | ❌ 概念专用 | ✅ 大盘+行业+概念统一 |
| 回测验证 | ✅ 77.3%已知 | ❌ 待跑 |

**结论：新设计在理论上更接近6.2原文的量价行为逻辑，但必须通过回测验证才能说"更科学"。**

## 六、回测方案

### 数据源

中证全指（000985）日K线，200条以上回测窗口。

### 回测方法

滚动窗口回测（类似signal_detector/backtest.py模式）：

```python
for i in range(80, len(klines)):
    result = judge_peak_valley_v2(klines[:i+1])
    if result['position'] == '偏波谷':
        # 测未来5/10/20日涨幅
        forward_return = (klines[min(i+5, len(klines)-1)]['close'] - klines[i]['close']) / klines[i]['close']
        stats.record('valley', forward_return)
    elif result['position'] == '偏波峰':
        forward_return = (klines[min(i+5, len(klines)-1)]['close'] - klines[i]['close']) / klines[i]['close']
        stats.record('peak', forward_return)
```

### 统计指标

| 指标 | 说明 |
|:----|:-----|
| 偏波谷后5日胜率 | 判波谷后涨的概率 |
| 偏波谷后5日均收益 | 判波谷后的平均收益 |
| 偏波峰后5日胜率 | 判波峰后跌的概率 |
| 偏波峰后5日均收益 | 判波峰后的平均收益 |
| 触发频率 | 每种信号出现的频率 |
| 按结构分段 | 上升/下降/震荡各结构下的胜率 |

### 基准对比

同时跑旧的 `judge_peak_valley()`（BIAS20版本）做A/B对比。

## 八、不做什么

1. **个股只加辅助字段，不做独立判定** — `get_stock_card()` 新增 `wave_position` 字段（波谷/波峰/中间），复用 `judge_structure_wave()` 结果。个股波谷70.6%可辅助买点确认，不做独立买卖信号。
2. 不做多周期并行判断（只用日线）
3. 不做ML预测（纯规则引擎）
4. 不改变现有调用的输出接口（兼容现有调用方）

## 九、回测结果

> 以下为v6最终版回测结果。数据源：374个概念板块×241个交易日，滚动窗口80天。

### 核心指标
