## 1. 产品定位

这是一套**辅助你筛选、聚焦、验证市场最强逻辑**的工具。核心理念：

> **不是先想好逻辑再录入，而是从日常阅读的资料中提炼逻辑，系统帮你分级、聚焦、自动验证。**

| 你做什么 | 系统做什么 |
|:---------|:----------|
| 读公众号/研报/视频 → 发现逻辑方向 | 投喂后自动提取摘要、打标签、归类 |
| 设定聚焦方向（1~3个） | 对聚焦逻辑自动走势验证 + 主动推送异常 |
| 录入未来事件日历 | 事件当天提醒，事件后自动验证预判对错 |
| 定期审视 + 手动修正系统打分 | 对所有逻辑自动验证 + 打分 + 升降级建议 |

**系统定位：** 辅助你构建最强逻辑股池的中间环节。**逻辑由你发现，跟踪由系统完成。**

## 3. 核心概念

### 3.1 逻辑标签

逻辑标签是系统核心。所有投喂的资料和前置事件都归入某个逻辑标签。

每个逻辑标签包含：名称、关联行业、关联个股、**层级（聚焦/核心/观察）**、事件数、印证率。

### 3.2 资料条目

每条你投喂的资料（链接/PDF/抖音）构成一个条目。系统自动归属到对应的逻辑标签。

### 3.3 前置预判条目

你手动录入的未来事件，和资料条目共享相同的标签体系，但多了 `event_date`（事件日期）和 `prediction`（你的预判）。

### 3.4 催化剂时间线

一个逻辑标签下，所有资料条目 + 前置预判条目按时间排列，构成"逻辑故事线"。

## 4. 逻辑分层与聚焦机制

### 4.1 三层分级

这是过滤的核心。逻辑不是平级的，系统帮你分层管理，让你聚焦在最值得关注的方向。

```
🌟 聚焦（1~3个）
  ├── 你当前最关注的大方向
  ├── 出现在首页看板顶部
  ├── 自动走势验证 + 关联个股异常主动推送
  └── 每日收盘后自动更新数据

📌 核心（不限）
  ├── 你确认过的中长期方向
  ├── 正常显示在列表中
  └── 自动走势验证，但不主动推送

👁️ 观察（不限）
  ├── 新投喂的资料自动归入此处
  ├── 列表折叠显示
  ├── 自动走势验证，但不主动推送
  └── 30天无新事件 → 提示归档
```

### 4.2 自动升降级建议

系统根据事件积累和印证率，主动建议升降级：

| 当前级别 → 目标 | 触发条件 | 系统提示 |
|:---------------|:---------|:---------|
| 观察 → 核心 | 累计3+事件 **且** 印证率≥60% | "此逻辑已积累3个事件且印证良好，建议升级为核心？" |
| 核心 → 观察 | 30天无新事件 **或** 最近3个事件印证率≤30% | "此逻辑已30天无新催化且最近事件印证较差，建议降级观察" |
| 核心 → 聚焦 | 符合核心条件 **且** 你手动设聚焦 | 你主动操作 |

**规则：** 系统只是**建议**，最终决定权在你。

### 4.3 聚焦管理

- 聚焦区始终只有1~3个逻辑
- 要把一个逻辑设为聚焦，如果已满3个，必须先取消一个
- 这强迫你"将军赶路，不追小兔"

## 5. 走势验证机制

### 5.1 自动验证（所有层级）

**不再需要你手动点按钮。** 系统在以下时机自动跑走势验证：

| 触发时机 | 验证内容 |
|:---------|:---------|
| 新资料投喂后 | 关联个股3/5/10日涨跌幅 + 板块排名变化 |
| 前置预判事件发生后 | 对比基线数据与事件后走势 |
| 每日收盘后（聚焦逻辑） | 关联个股每日涨跌 + 买点信号变化 |
| 每周收盘后（全部逻辑） | 整体印证率重新计算 |

### 5.2 验证结果输出

每条验证结果：

```
📊 走势验证
  ├── 关联个股信息（复用 get_stock_card，含买点信号）
  ├── 板块排名变化（如：封测 #5 → #3 ↑）
  ├── 买点信号数量变化
  └── 📝 文字总结（AI生成）
      "事件后3天关联个股平均涨+5.2%，板块排名上升，逻辑印证良好"
```

### 5.3 系统打分

系统自动给出评分：

| 分数 | 条件 |
|:----|:-----|
| ✅ 印证 | 关联个股平均涨幅 > 大盘 **且** 板块排名上升 |
| ➖ 一般 | 关联个股走势与大盘持平 **或** 板块排名不变 |
| ❌ 背离 | 关联个股平均跑输大盘 **或** 板块排名下降 |

### 5.4 人工修正

你的判断**优先级高于系统打分**。你可以随时手动修改一条验证的评分。

## 6. 前置预判（事件日历）

### 6.1 出发点

原文提到：
> "对于逻辑的起爆点，可以做一些前置分析，比如下个月有苹果的发布会。前置分析，可以让我们提前去潜伏一个板块。"

现在的设计是被动的（事件发生后你投喂→验证），但还可以**主动前瞻**。

### 6.2 使用场景

```
你可能在微信群里看到一张图：
"6月重大事件日历"
├── 6/03 英伟达财报
├── 6/10 苹果WWDC
├── 6/15 美联储议息
├── 6/20 华为新品发布会
└── 6/25 某行业峰会

你把这张图里的几个关键事件录入系统：
  事件：英伟达FY26Q2财报
  日期：2026-06-03
  关联逻辑：AI算力链条
  关联个股：中际旭创、天孚通信、工业富联
  你预判：超预期→利好光模块
```

### 6.3 处理流程

```
你录入预期事件
  ↓
系统记录基线（当前价格、板块排名）
  ↓
事件前N天（可设3天/7天）→ 提醒你关注
  ↓
事件当天 → 提醒"今天是英伟达财报日"
  ↓
事件后自动验证（3/5/10天）
  ├── 关联个股涨跌幅 vs 基线
  ├── 板块排名变化
  └── 对比你的预判 → "预判正确 / 预判错误"
  ↓
你审视预判结果
```

### 6.4 两种模式

| 模式 | 说明 | 阶段 |
|:----|:-----|:----|
| **L1：你手动给日历** | 你给我一份事件日程，录入系统 | P1 |
| **L2：系统自动抓** | 从财联社等自动获取事件日历 | P3 |

## 7. 功能模块

|:----|:-----|

| 层级设置 | 手动设为聚焦/核心/观察 |

| 功能 | 说明 |
|:----|:-----|

| 上传PDF | 研报PDF，自动解析文字+提取摘要 |
| 投喂抖音 | 粘贴抖音链接 → 调用提取程序 |
| 自动处理 | 提取摘要 + 打行业/公司/逻辑标签 |
| 预览确认 | 编辑标签后再保存 |

### 7.3 走势验证（P0 ~ P1）

| 功能 | 说明 |
|:----|:-----|
| 自动验证 | 新事件投喂后自动跑，无需手动 |
| 聚焦逻辑主动推送 | 关联个股异常时提醒 |
| 系统打分 | 自动判断印证/一般/背离 |
| 人工修正 | 你可修改系统打分，优先级更高 |
| 验证历史 | 每个事件都有验证记录 |

### 7.4 前置预判（P1）

| 功能 | 说明 |
|:----|:-----|
| 录入预期事件 | 事件名+日期+关联逻辑+个股+预判 |
| 基线记录 | 录入时自动保存当前价格/排名 |
| 事件提醒 | 事件前/当天系统提醒 |
| 自动验证 | 事件后自动跑走势对比 |
| 预判对错统计 | 累计"预判正确率" |

### 7.5 搜索（P1）

按逻辑标签筛选 + 关键词搜索

### 7.6 逻辑时间线（P1）

按时间排列所有事件（资料+前置预判），展示走势验证结果

## 11. 逻辑失效判断机制

双轨制：

```
系统自动检测
  ├── 关联个股连续跑输大盘 → 提示
  ├── 板块排名持续下滑 → 提示
  ├── 最近3个事件印证率≤30% → 建议降级
  └── 30天无新事件 → 提示归档

用户手动标记（优先级更高）
  └── 系统建议降级 → 你可以"忽略，保持活跃"
```
