## 1. 产品定位

这是一套**辅助你筛选、聚焦、验证市场最强逻辑**的工具。核心理念：

> **不是先想好逻辑再录入，而是从日常阅读的资料中提炼逻辑，系统帮你分级、聚焦、自动验证。**

| 你做什么 | 系统做什么 |
|:---------|:----------|
| 读公众号/研报/视频 → 发现逻辑方向 | 投喂后自动提取摘要、打标签、归类 |
| 设定聚焦方向（1~3个） | 对聚焦逻辑自动走势验证 + 主动推送异常 |
| 录入未来事件日历 | 事件当天提醒，事件后自动验证预判对错 |
| 定期审视 + 手动修正系统打分 | 对所有逻辑自动验证 + 打分 + 升降级建议 |
| 查看最强逻辑股池报告 | 自动生成股池列表 + 每只股的选择依据 |

**系统定位：** 辅助你构建最强逻辑股池的中间环节。**逻辑由你发现，跟踪由系统完成。**

## 3. 核心概念

### 3.1 逻辑标签

逻辑标签是系统核心。所有投喂的资料和前置事件都归入某个逻辑标签。

每个逻辑标签包含：名称、关联行业、关联个股、**层级（聚焦/核心/观察）**、事件数、走势印证率、业绩印证率。

### 3.2 资料条目

每条投喂的资料（链接/PDF/抖音）构成一个条目。系统自动归属到对应的逻辑标签。

### 3.3 前置预判条目

手动录入的未来事件。有多了一个 `event_date` 和 `prediction`。

### 3.4 催化剂时间线

一个逻辑标签下，所有资料条目 + 前置预判条目按时间排列，构成"逻辑故事线"。

### 3.5 标签匹配算法

系统投喂资料后，需要自动匹配已有逻辑标签。采用**混合策略（方案C）**：

### 方案A：关键词匹配（粗筛）

```
文章摘要中扫描逻辑标签的关联关键词
  ├── 标签名称关键词："先进封装" "封测" "封装"
  ├── 关联行业关键词："半导体" "PCB"
  └── 关联公司关键词："长电科技" "600584"

→ 命中 → 推荐该标签并给出匹配度
→ 未命中 → 进入方案B
```

- ✅ 快（秒级），零成本
- ❌ 同义词/语义不匹配时不灵
- 用途：第一道快速筛，减少LLM调用次数

### 方案B：LLM语义匹配（精筛）

```
将文章摘要 + 已有逻辑标签列表 发给LLM
  ↓
LLM返回：
  匹配标签：
    1. 先进封装加速（匹配度92%）← 推荐
    2. AI算力链条（匹配度45%）
  或：未匹配到已有标签，建议新建
```

- ✅ 能理解语义，同义词（chiplet=先进封装）
- ❌ 有API调用成本，速度~1-2秒
- 用途：关键词未命中时兜底

### 方案C：混合策略（P0实施）

```
文章投喂后
  ↓
① 关键词粗筛（方案A）
  ├── 命中 → 推荐匹配标签 + 匹配度
  └── 未命中或匹配度低 → 
        ↓
② LLM语义匹配（方案B）
  ├── 匹配到已有标签 → 推荐
  └── 全部不匹配 → 建议新建标签
  ↓
③ 预览确认环节，你可调整/修改
```

### 3.6 最强逻辑股池

由聚焦逻辑+核心逻辑下的关联个股组成，系统自动聚合每只股的：
- 选择理由（隶属于哪个逻辑）
- 相关事件/研报
- 走势验证结果
- 业绩验证结果
- 当前买点信号

### 3.7 逻辑验证的两重自动扫描

系统会持续从全市场数据中自动扫描，为逻辑验证注入素材：

| 扫描 | 频率 | 核心作用 |
|:----|:-----|:---------|
| 🔴 **每日涨停扫描** | 每日收盘后 | **早发现** — 涨停→匹配已有逻辑→验证通过 |
| 🔵 **每日新高扫描** | 每日收盘后 | **强验证** — 新高→匹配已有逻辑→验证加分 |

**组合链路：涨停发现 → 入库逻辑 → 新高验证 → 逻辑健康分提升**

同板块/同逻辑的个股连续涨停是逻辑验证的最强信号之一，而新高则是逻辑已被市场定价的最可靠证据。

> 📌 **同花顺热点复盘：** 涨停热点复盘图（如每日板块涨停家数分布）可由你手动补充，系统作为辅助参考信息在逻辑股池中展示。

## 4. 逻辑分层与聚焦机制

### 4.1 三层分级

逻辑不是平级的，系统帮你分层聚焦。

```
🌟 聚焦（1~3个）
  ├── 你当前最关注的大方向
  ├── 出现在首页看板顶部
  ├── 自动走势验证 + 关联个股异常主动推送
  ├── 每日收盘后自动更新数据
  └── 业绩发布时主动验证

📌 核心（不限）
  ├── 你确认过的中长期方向
  ├── 正常显示在列表中
  └── 自动走势验证但不主动推送

👁️ 观察（不限）
  ├── 新投喂的资料自动归入此处
  ├── 列表折叠显示
  ├── 自动走势验证但不主动推送
  └── 30天无新事件 → 提示归档
```

### 4.2 自动升降级建议

| 当前 → 目标 | 触发条件 | 系统提示 |
|:------------|:---------|:---------|
| 观察 → 核心 | 累计3+事件 **且** 走势印证率≥60% | "建议升级为核心？" |
| 核心 → 观察 | 30天无新事件 **或** 最近3个事件印证率≤30% | "建议降级观察" |
| 核心 → 聚焦 | 你手动操作 | — |

### 4.3 聚焦上限

- 聚焦区仅1~3个
- 满3个时需取消一个才能加新的
- 强迫"将军赶路，不追小兔"

## 5. 走势验证机制

### 5.1 自动验证（所有层级）

系统自动跑，不再需要手动点按钮。

| 触发时机 | 验证内容 |
|:---------|:---------|
| 新资料投喂后 | 关联个股3/5/10日涨跌幅 + 板块排名变化 |
| 前置预判事件发生后 | 对比基线数据与事件后走势 |
| 每日收盘后（聚焦逻辑） | 关联个股每日涨跌 + 买点信号变化 |
| 每周收盘后（全部逻辑） | 整体印证率重新计算 |

### 5.2 验证结果输出

```
📊 走势验证
  ├── 关联个股信息（复用 get_stock_card，含买点信号）
  ├── 板块排名变化（如：封测 #5 → #3 ↑）
  ├── 买点信号数量变化
  └── 📝 文字总结（AI生成）
      "事件后3天关联个股平均涨+5.2%，板块排名上升，逻辑印证良好"
```

### 5.3 系统打分

| 分数 | 条件 |
|:----|:-----|
| ✅ 印证 | 关联个股平均涨幅 > 大盘 **且** 板块排名上升 |
| ➖ 一般 | 关联个股走势与大盘持平 **或** 板块排名不变 |
| ❌ 背离 | 关联个股平均跑输大盘 **或** 板块排名下降 |

### 5.4 人工修正

你的判断**优先级高于系统打分**。可随时修改。

### 5.5 🔴 每日涨停扫描（全自动）

#### 5.5.1 数据源

来自 `akshare.stock_zt_pool_em(date)`，返回当日沪深两市所有涨停股，含：

| 字段 | 说明 |
|:-----|:-----|
| 股票代码、名称 | 基础信息 |
| 涨跌幅 | 通常≈10% |
| 首次封板时间 | HHMMSS格式，早封=强 |
| 连板数 | 1板/2板/3板+，连板越多信号越强 |
| 所属行业 | 同花顺行业分类 |
| 封板资金 | 封单金额 |
| 成交额/换手率 | 量能数据 |

#### 5.5.2 处理流程

```
每日收盘后（~16:00）
  ↓
拉取当日涨停板全量（~50-100只）
  ↓
对每只涨停股：
  ├─ 匹配已有逻辑标签（方案C匹配算法）
  │   ├── 命中 → 记录为「走势验证通过」事件
  │   │   ├── 首次封板早 = 加分
  │   │   ├── 连板数≥2 = 强信号
  │   │   └── 同逻辑多只涨停 = 最强信号
  │   └── 未匹配 → 进入「候选逻辑推荐」
  │       ├── LLM提取关键词 → 推荐候选逻辑标签
  │       └── 投放预览区，等你手动确认
  ↓
输出「涨停股池逻辑分布」（可选展示在逻辑股池页）
```

#### 5.5.3 涨停验证的三种场景

| 场景 | 信号强度 | 对逻辑健康的影响 |
|:-----|:--------:|:----------------|
| 逻辑关联个股**首次涨停** | ⭐ | 逻辑短期被市场认可，健康分+1 |
| 逻辑关联个股**连板≥2** | ⭐⭐ | 持续性强，健康分+2 |
| 同逻辑**多只个股同日涨停** | ⭐⭐⭐ | 板块效应形成，健康分+3，推送提醒 |

#### 5.5.4 噪音过滤

涨停中既有逻辑驱动的也有事件驱动的。「每日涨停扫描」的作用是**广撒网发现线索**，不要求每只涨停都有逻辑关联。

- 单次涨停 → 仅记录，不推送
- 连板≥2 或 同板块多只涨停 → 推送提醒
- 后续走势（是否新高、是否持续性）自然过滤噪音

### 5.6 🔵 每日新高扫描（全自动）

#### 5.6.1 数据源

来自 `akshare.stock_rank_cxg_ths()`，当日创历史新高的全市场个股列表。

#### 5.6.2 处理流程

```
每日收盘后（~16:00）
  ↓
拉取当日新高股池
  ↓
对每只新高股：
  ├─ 匹配已有逻辑标签（方案C匹配算法）
  │   ├── 命中 → 记录为「走势验证通过」事件，健康分+3
  │   └── 未匹配 → 推荐候选标签存入预览区
  ↓
输出「新高股池逻辑分布」
```

#### 5.6.3 涨停 vs 新高：两条互补的验证线

| 维度 | 🔴 涨停扫描 | 🔵 新高扫描 |
|:----|:-----------|:-----------|
| 信号性质 | 短期爆发力 | 中期趋势定价 |
| 价值 | **早发现** — 逻辑刚被市场认可 | **强验证** — 逻辑已被充分定价 |
| 噪音 | 高（游资/事件驱动） | 低（机构/趋势资金） |
| 对逻辑健康影响 | +1~3 | +3 |
| 推送策略 | 连板≥2或板块联动才推 | 匹配即推（信号可靠）

## 6. 业绩验证机制

### 6.1 为什么要做业绩验证

> **"逻辑的终点，是业绩。所有的牛股，都和业绩有关。没有业绩的逻辑，炒作可以，但大概率哪里来回到哪里去。"**

走势验证看的是**市场情绪**（股价涨跌），但股价涨有可能是炒作。业绩验证看的是**逻辑到底有没有真的赚钱**。

### 6.2 触发时机

| 触发时机 | 验证内容 |
|:---------|:---------|
| 关联公司发布**业绩预告** | 抓取预告数据，对比市场预期 |
| 关联公司发布**正式季报/年报** | 营收增速、利润增速、毛利率 |
| 关联公司发布**业绩修正公告** | 上修→加分，下修→警示 |

### 6.3 验证流程

```
聚焦逻辑"先进封装加速"
  关联公司：长电科技、通富微电、华峰测控
     ↓
6月中旬 长电科技发布中报预告
     ↓
系统自动抓取业绩数据
  ├── 营收同比：+35%
  ├── 扣非利润同比：+50%
  └── 是否超预期：是（市场预期+30%）
     ↓
结论："中报预告利润+50%超预期 → 业绩印证逻辑 ✅"
```

### 6.4 输出

```
📋 业绩验证
  ├── 公司：长电科技（600584）
  ├── 事件：2026年中报业绩预告
  ├── 营收：+35% (超预期)
  ├── 利润：+50% (超预期)
  ├── 结论：业绩印证逻辑 ✅
  └── 📝 文字总结
```

### 6.5 与走势验证的关系

两条验证线**共同决定逻辑健康度**：

```
逻辑健康度
  ├── 📊 走势印证率（50%）
  │   └── 市场情绪层面
  └── 📋 业绩印证率（50%）
      └── 基本面层面

最佳：走势✅ + 业绩✅  → 逻辑健康
危险：走势✅ + 业绩❌  → 纯炒作，警惕
```

### 6.6 关联前置预判

业绩发布是**可预知的**。前置预判录入财报日期后，事件当天自动触发：
1. ✅ 走势验证（股价反应）
2. ✅ 业绩验证（财务数据）
全部自动完成。

## 7. 前置预判（事件日历）

### 7.1 出发点

> "对于逻辑的起爆点，可以做一些前置分析，比如下个月有苹果的发布会。前置分析，可以让我们提前去潜伏一个板块。"

### 7.2 使用场景

```
微信群里看到"6月重大事件日历"
├── 6/03 英伟达财报
├── 6/10 苹果WWDC
├── 6/15 美联储议息
├── 6/20 华为新品发布会
└── 6/25 某行业峰会

录入关键事件：
  事件：英伟达FY26Q2财报
  日期：2026-06-03
  关联逻辑：AI算力链条
  关联个股：中际旭创、天孚通信
  你的预判：超预期→利好光模块
```

### 7.3 处理流程

```
录入事件 → 记录基线（价格/排名）
  → 事件前N天提醒
  → 事件当天提醒
  → 事件后自动验证（走势+业绩）
  → 对比预判 → "正确/错误"
```

### 7.4 两种模式

| 模式 | 说明 | 阶段 |
|:----|:-----|:----|
| L1：你手动给日历 | 粘贴事件日程录入 | P1 |
| L2：系统自动抓 | 从财联社等获取 | P3 |

## 8. 最强逻辑股池与报告

### 8.1 什么是最强逻辑股池

简放的三个股池体系：
| 股池 | 方式 | 用途 |
|:----|:----|:-----|
| 动量股池 | 系统全市场扫描 | 感知资金方向 |
| 新高股池 | 系统每日自动 | 判断市场强弱 |
| **最强逻辑股池** | **你分析逻辑后选股** | **核心持仓候选** |

我们的最强逻辑追踪系统，最终产出就是这个股池。

每日涨停扫描和新高扫描是支撑最强逻辑股池的两大自动验证引擎：

```
涨停扫描 → 发现逻辑信号 → 
                              → 验证已有逻辑 → 更新健康分 → 最强逻辑股池
新高扫描 → 确认逻辑定价 → 
                                            ↘ 发现新逻辑 → 推荐候选 → 手动确认
```

### 8.2 股池生成规则

```
聚焦逻辑 + 核心逻辑
  ↓
提取所有关联个股 → 去重
  ↓
为每只股票组装：
  ├── 所属逻辑（为什么选它）
  ├── 相关事件/研报列表
  ├── 走势验证结果（近30日涨跌幅）
  ├── 业绩验证结果（最新财报数据）
  └── 当前买点信号（如有）
```

### 8.3 股池展示

```
🌟 最强逻辑股池（2026-05-27）

┌─ 逻辑：先进封装加速（聚焦） ───────────────┐

│                                              │

│  ├── 选择理由：先进封装产能扩张             │
│  └── 业绩：中报预告利润+30%                │
└──────────────────────────────────────────────┘
```

### 8.4 股池报告

可一键生成 **最强逻辑股池周报**，包含：

| 内容 | 说明 |
|:----|:-----|
| 当前聚焦逻辑概览 | 名称、状态、事件数、印证率 |
| 每只股的详细资料 | 选择理由、走势、业绩、研报、买点 |
| 总结建议 | "当前3个聚焦逻辑均被业绩验证，可重点关注" |

报告格式：HTML页面（可在浏览器查看）或 Markdown。

## 12. 信息源与处理流程

### 12.1 公众号文章 / 新闻链接

粘贴链接 → 抓取 → 提取摘要 → 打标签 → 预览确认 → 保存 → 自动走势验证

### 12.2 研报 PDF

上传 → 解析 → 提取摘要 → 打标签 → 预览确认 → 保存原文 → 自动走势验证

### 12.3 抖音视频

粘贴链接 → 调用提取程序 → 编辑 → 保存 → 自动走势验证（P2）

### 12.4 前置预判

L1：手动录入 → 基线 → 提醒 → 自动验证（P1）
L2：系统自动抓日历（P3）

### 12.5 业绩验证

聚焦/核心逻辑关联公司发布财报 → 自动抓取数据 → 对比预期 → 更新印证率（P1）

### 12.6 🔴 每日涨停扫描（全自动，P0）

每日收盘后自动拉取涨停板池，匹配已有逻辑标签。匹配成功率+3，未匹配推荐候选。

### 12.7 🔵 每日新高扫描（全自动，P0）

每日收盘后自动拉取新高股池，匹配已有逻辑标签。匹配成功逻辑健康分+3，未匹配推荐候选。

### 12.8 最强逻辑股池

从聚焦+核心逻辑提取个股 → 组装完整信息 → 生成报告（P1~P2）

## 13. 逻辑失效判断机制

```
系统自动检测
  ├── 关联个股连续跑输大盘 → 提示
  ├── 板块排名持续下滑 → 提示
  ├── 最近3个事件印证率≤30% → 建议降级
  ├── 最新业绩不及预期 → 警示
  ├── 走势✅但业绩❌ → 警惕纯炒作
  └── 30天无新事件 → 提示归档

用户手动标记（优先级更高）
  └── 系统建议降级 → 可"忽略，保持活跃"
```

## 16. 推送通知机制

聚焦逻辑的异常提醒，支持三种推送方式：

| 方式 | 目标 | 说明 | 阶段 |
|:----|:-----|:-----|:----|
| 🔔 页面内报警池 | 盯盘页面报警区 | 复用现有报警系统，新增一条报警记录 | P1 |
| 💬 微信私信 | 你的微信 | 系统直接发送消息到你的微信 | P1 |
| 👥 微信群 | 微信群组 | 推送到注册的微信群 | P2 |

### 推送触发场景

| 触发条件 | 推送内容 |
|:---------|:---------|
| 聚焦逻辑关联个股集体走弱 | "先进封装加速关联个股连续3天跑输大盘" |
| 聚焦逻辑关联公司发布业绩预告 | "长电科技发布中报预告，利润+50%超预期" |
| 前置预判事件当天提醒 | "提醒：今天英伟达财报日" |
| 系统建议升降级 | "先进封装加速已累积4个事件，建议升级为聚焦" |
