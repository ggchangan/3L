## 1. 产品定位

这是一套**辅助你筛选、聚焦、验证市场最强逻辑**的工具。核心理念：

> **不是先想好逻辑再录入，而是从日常阅读的资料中提炼逻辑，系统帮你分级、聚焦、自动验证。**

| 你做什么 | 系统做什么 |
|:---------|:----------|
| 读公众号/研报/视频 → 发现逻辑方向 | 投喂后自动提取摘要、打标签、归类 |
| 设定聚焦方向（1~3个） | 对聚焦逻辑自动走势验证 + 主动推送异常 |
| 录入未来事件日历 | 事件当天提醒，事件后自动验证预判对错 |
| 定期审视 + 手动修正系统打分 | 对所有逻辑自动验证 + 打分 + 升降级建议 |
| 查看最强逻辑股池报告 | 自动生成股池列表 + 每只股的选择依据 |

**系统定位：** 辅助你构建最强逻辑股池的中间环节。**逻辑由你发现，跟踪由系统完成。**

## 3. 核心概念

### 3.1 逻辑标签

逻辑标签是系统核心。所有投喂的资料和前置事件都归入某个逻辑标签。

每个逻辑标签包含：名称、关联行业、关联个股、**层级（聚焦/核心/观察）**、事件数、走势印证率、业绩印证率。

### 3.2 资料条目

每条投喂的资料（链接/PDF/抖音）构成一个条目。系统自动归属到对应的逻辑标签。

### 3.3 前置预判条目

手动录入的未来事件。有多了一个 `event_date` 和 `prediction`。

### 3.4 催化剂时间线

一个逻辑标签下，所有资料条目 + 前置预判条目按时间排列，构成"逻辑故事线"。

### 3.5 标签匹配算法

系统投喂资料后，需要自动匹配已有逻辑标签。采用**混合策略（方案C）**：

### 方案A：关键词匹配（粗筛）

```
文章摘要中扫描逻辑标签的关联关键词
  ├── 标签名称关键词："先进封装" "封测" "封装"
  ├── 关联行业关键词："半导体" "PCB"
  └── 关联公司关键词："长电科技" "600584"

→ 命中 → 推荐该标签并给出匹配度
→ 未命中 → 进入方案B
```

- ✅ 快（秒级），零成本
- ❌ 同义词/语义不匹配时不灵
- 用途：第一道快速筛，减少LLM调用次数

### 方案B：LLM语义匹配（精筛）

```
将文章摘要 + 已有逻辑标签列表 发给LLM
  ↓
LLM返回：
  匹配标签：
    1. 先进封装加速（匹配度92%）← 推荐
    2. AI算力链条（匹配度45%）
  或：未匹配到已有标签，建议新建
```

- ✅ 能理解语义，同义词（chiplet=先进封装）
- ❌ 有API调用成本，速度~1-2秒
- 用途：关键词未命中时兜底

### 方案C：混合策略（P0实施）

```
文章投喂后
  ↓
① 关键词粗筛（方案A）
  ├── 命中 → 推荐匹配标签 + 匹配度
  └── 未命中或匹配度低 → 
        ↓
② LLM语义匹配（方案B）
  ├── 匹配到已有标签 → 推荐
  └── 全部不匹配 → 建议新建标签
  ↓
③ 预览确认环节，你可调整/修改
```

### 3.6 最强逻辑股池

由聚焦逻辑+核心逻辑下的关联个股组成，系统自动聚合每只股的：
- 选择理由（隶属于哪个逻辑）
- 相关事件/研报
- 走势验证结果
- 业绩验证结果
- 当前买点信号

### 3.7 逻辑验证的两重自动扫描

系统会持续从全市场数据中自动扫描，为逻辑验证注入素材：

| 扫描 | 频率 | 核心作用 |
|:----|:-----|:---------|
| 🔴 **每日涨停扫描** | 每日收盘后 | **早发现** — 涨停→匹配已有逻辑→验证通过 |
| 🔵 **每日新高扫描** | 每日收盘后 | **强验证** — 新高→匹配已有逻辑→验证加分 |

**组合链路：涨停发现 → 入库逻辑 → 新高验证 → 逻辑健康分提升**

同板块/同逻辑的个股连续涨停是逻辑验证的最强信号之一，而新高则是逻辑已被市场定价的最可靠证据。

> 📌 **同花顺热点复盘：** 涨停热点复盘图（如每日板块涨停家数分布）可由你手动补充，系统作为辅助参考信息在逻辑股池中展示。

### 3.8 自动发现新逻辑候选

涨停/新高扫描「未匹配」的结果并非浪费——它们是**市场正在定价但系统尚未追踪的信号**。

```
涨停/新高扫描
  ↓
未匹配到已有逻辑标签的个股
  ↓
LLM提取共同特征（行业/概念/关键词/事件驱动类型）
  ↓
发现共性：3只PCB股同日涨停 → 自动生成「PCB景气度反转」候选标签
  ↓
放入预览区 → 你确认或忽略
```

**价值：** 你不需要一开始就想好所有逻辑方向。系统会从市场信号中帮你捕捉可能的新逻辑，你来判断是否值得追踪。

### 3.9 逻辑关联图谱

市场逻辑不是孤立的，它们之间存在天然的关联关系。通过持续投喂资料，系统自动将每篇文章/事件拆解为 {事件 → 逻辑 → 产业 → 个股} 的链条，多条链条叠加形成完整的逻辑图谱。

#### 3.9.1 链式拆解

每次投喂系统自动提取一条完整链条：

```
📄 文章/事件
   ↓
🎯 逻辑（华为τ定律→先进封装升级）
   ↓
🏭 产业（先进封装、EDA、精密零部件）
   ↓
📈 个股（长电科技600584、中际旭创300502）
```

#### 3.9.2 图谱叠加

多篇文章投喂后，链条自然叠加成网络：

```
  文章A ──→ 华为τ定律   ──→ 先进封装 ──→ 长电科技(600584)
  文章B ──→ AI算力需求  ──→ 光模块   ──→ 中际旭创(300502)
  文章C ──→ 英伟达财报  ──→ PCB      ──→ 沪电股份(002463)
                                    ↓
  文章D ──→ 先进封装景气 ──→ 先进封装 ──→ 长电科技(600584)
                                           通富微电(002156)
```

从图谱中可以快速识别：
- **核心逻辑** — 被多篇文章印证、关联多只个股的逻辑
- **核心个股** — 被多个逻辑指向、多次验证通过的个股
- **产业联动** — 上游逻辑验证通过 → 下游逻辑自动加分

#### 3.9.3 关系类型

| 关系类型 | 示例 | 设计影响 |
|:---------|:-----|:---------|
| **上下游** | AI算力 → 光模块 → 先进封装 → PCB | 上游逻辑验证 → 下游逻辑加分 |
| **同板块** | 先进封装、芯片设备、EDA | 板块动量共享 |
| **竞争/替代** | 硅光 vs 传统光模块 | 一个逻辑弱化，另一个可能增强 |
| **互补** | 算力基建 + 电力需求 | 两者共同作用推动主线 |

#### 3.9.4 数据模型

```json
{
  "id": "tag-ai-chain",
  "name": "AI算力链条",
  "relations": [
    {"type": "downstream", "target_id": "tag-optical", "strength": "strong"},
    {"type": "upstream", "target_id": "tag-advanced-pkg", "strength": "medium"}
  ]
}
```

**设计影响：**
- 当某个逻辑被涨停/新高验证→关联逻辑健康分也自动提升（按关系强度加权）
- 关联图谱在逻辑详情页可视化展示
- 初步从简单关系开始（上下游+同板块），后续可再扩展

### 3.10 资金面辅助验证

在走势+业绩双验证之外，增加第三个验证维度——**资金面验证**。

#### 3.10.1 数据源

已有系统数据：同花顺行业板块实时排名 `stock_board_industry_summary_ths()` 包含板块资金净流入。

#### 3.10.2 验证方式

```
逻辑关联板块的{
  └── 今日资金净流入金额
  └── 近5日累计净流入趋势
  └── 领涨股涨跌幅
}
  ↓
走势✅ + 资金流入✅ → 强势逻辑确认
走势✅ + 资金流出❌ → 警惕拉高出货
走势❌ + 资金流入✅ → 可能有资金暗中布局
走势❌ + 资金流出❌ → 逻辑弱化
```

#### 3.10.3 三种验证维度的关系

```
逻辑健康度
  ├── 📊 走势印证率（40%） — 股价涨跌
  ├── 📋 业绩印证率（40%） — 基本面
  └── 💰 资金印证率（20%） — 资金态度

红色预警：走势❌ + 资金❌ → 立即审视
快速警戒：走势❌ + 资金✅ → 观察不操作
最优组合：走势✅ + 业绩✅ + 资金✅ → 最强势逻辑
```

## 4. 逻辑分层与聚焦机制

### 4.1 三层分级

逻辑不是平级的，系统帮你分层聚焦。

```
🌟 聚焦（1~3个）
  ├── 你当前最关注的大方向
  ├── 出现在首页看板顶部
  ├── 自动走势验证 + 关联个股异常主动推送
  ├── 每日收盘后自动更新数据
  └── 业绩发布时主动验证

📌 核心（不限）
  ├── 你确认过的中长期方向
  ├── 正常显示在列表中
  └── 自动走势验证但不主动推送

👁️ 观察（不限）
  ├── 新投喂的资料自动归入此处
  ├── 列表折叠显示
  ├── 自动走势验证但不主动推送
  └── 30天无新事件 → 提示归档
```

### 4.2 自动升降级建议

| 当前 → 目标 | 触发条件 | 系统提示 |
|:------------|:---------|:---------|
| 观察 → 核心 | 累计3+事件 **且** 走势印证率≥60% | "建议升级为核心？" |
| 核心 → 观察 | 30天无新事件 **或** 最近3个事件印证率≤30% | "建议降级观察" |
| 核心 → 聚焦 | 你手动操作 | — |

### 4.3 聚焦上限

- 聚焦区仅1~3个
- 满3个时需取消一个才能加新的
- 强迫"将军赶路，不追小兔"

## 5. 走势验证机制

### 5.1 自动验证（所有层级）

系统自动跑，不再需要手动点按钮。

| 触发时机 | 验证内容 |
|:---------|:---------|
| 新资料投喂后 | 关联个股3/5/10日涨跌幅 + 板块排名变化 |
| 前置预判事件发生后 | 对比基线数据与事件后走势 |
| 每日收盘后（聚焦逻辑） | 关联个股每日涨跌 + 买点信号变化 |
| 每周收盘后（全部逻辑） | 整体印证率重新计算 |

### 5.2 验证结果输出

```
📊 走势验证
  ├── 关联个股信息（复用 get_stock_card，含买点信号）
  ├── 板块排名变化（如：封测 #5 → #3 ↑）
  ├── 买点信号数量变化
  └── 📝 文字总结（AI生成）
      "事件后3天关联个股平均涨+5.2%，板块排名上升，逻辑印证良好"
```

### 5.3 系统打分

| 分数 | 条件 |
|:----|:-----|
| ✅ 印证 | 关联个股平均涨幅 > 大盘 **且** 板块排名上升 |
| ➖ 一般 | 关联个股走势与大盘持平 **或** 板块排名不变 |
| ❌ 背离 | 关联个股平均跑输大盘 **或** 板块排名下降 |

### 5.4 人工修正

你的判断**优先级高于系统打分**。可随时修改。

### 5.5 🔴 每日涨停扫描（全自动）

#### 5.5.1 数据源

来自 `akshare.stock_zt_pool_em(date)`，返回当日沪深两市所有涨停股，含：

| 字段 | 说明 |
|:-----|:-----|
| 股票代码、名称 | 基础信息 |
| 涨跌幅 | 通常≈10% |
| 首次封板时间 | HHMMSS格式，早封=强 |
| 连板数 | 1板/2板/3板+，连板越多信号越强 |
| 所属行业 | 同花顺行业分类 |
| 封板资金 | 封单金额 |
| 成交额/换手率 | 量能数据 |

#### 5.5.2 处理流程

```
每日收盘后（~16:00）
  ↓
拉取当日涨停板全量（~50-100只）
  ↓
对每只涨停股：
  ├─ 匹配已有逻辑标签（方案C匹配算法）
  │   ├── 命中 → 记录为「走势验证通过」事件
  │   │   ├── 首次封板早 = 加分
  │   │   ├── 连板数≥2 = 强信号
  │   │   └── 同逻辑多只涨停 = 最强信号
  │   └── 未匹配 → 进入「候选逻辑推荐」
  │       ├── LLM提取关键词 → 推荐候选逻辑标签
  │       └── 投放预览区，等你手动确认
  ↓
输出「涨停股池逻辑分布」（可选展示在逻辑股池页）
```

#### 5.5.3 涨停验证的三种场景

| 场景 | 信号强度 | 对逻辑健康的影响 |
|:-----|:--------:|:----------------|
| 逻辑关联个股**首次涨停** | ⭐ | 逻辑短期被市场认可，健康分+1 |
| 逻辑关联个股**连板≥2** | ⭐⭐ | 持续性强，健康分+2 |
| 同逻辑**多只个股同日涨停** | ⭐⭐⭐ | 板块效应形成，健康分+3，推送提醒 |

#### 5.5.4 噪音过滤

涨停中既有逻辑驱动的也有事件驱动的。「每日涨停扫描」的作用是**广撒网发现线索**，不要求每只涨停都有逻辑关联。

- 单次涨停 → 仅记录，不推送
- 连板≥2 或 同板块多只涨停 → 推送提醒
- 后续走势（是否新高、是否持续性）自然过滤噪音

### 5.6 🔵 每日新高扫描（全自动）

#### 5.6.1 数据源

来自 `akshare.stock_rank_cxg_ths()`，当日创历史新高的全市场个股列表。

#### 5.6.2 处理流程

```
每日收盘后（~16:00）
  ↓
拉取当日新高股池
  ↓
对每只新高股：
  ├─ 匹配已有逻辑标签（方案C匹配算法）
  │   ├── 命中 → 记录为「走势验证通过」事件，健康分+3
  │   └── 未匹配 → 推荐候选标签存入预览区
  ↓
输出「新高股池逻辑分布」
```

#### 5.6.3 涨停 vs 新高：两条互补的验证线

| 维度 | 🔴 涨停扫描 | 🔵 新高扫描 |
|:----|:-----------|:-----------|
| 信号性质 | 短期爆发力 | 中期趋势定价 |
| 价值 | **早发现** — 逻辑刚被市场认可 | **强验证** — 逻辑已被充分定价 |
| 噪音 | 高（游资/事件驱动） | 低（机构/趋势资金） |
| 对逻辑健康影响 | +1~3 | +3 |
| 推送策略 | 连板≥2或板块联动才推 | 匹配即推（信号可靠）

## 6. 业绩验证机制

### 6.1 为什么要做业绩验证

> **"逻辑的终点，是业绩。所有的牛股，都和业绩有关。没有业绩的逻辑，炒作可以，但大概率哪里来回到哪里去。"**

走势验证看的是**市场情绪**（股价涨跌），但股价涨有可能是炒作。业绩验证看的是**逻辑到底有没有真的赚钱**。

### 6.2 触发时机

| 触发时机 | 验证内容 |
|:---------|:---------|
| 关联公司发布**业绩预告** | 抓取预告数据，对比市场预期 |
| 关联公司发布**正式季报/年报** | 营收增速、利润增速、毛利率 |
| 关联公司发布**业绩修正公告** | 上修→加分，下修→警示 |

### 6.3 验证流程

```
聚焦逻辑"先进封装加速"
  关联公司：长电科技、通富微电、华峰测控
     ↓
6月中旬 长电科技发布中报预告
     ↓
系统自动抓取业绩数据
  ├── 营收同比：+35%
  ├── 扣非利润同比：+50%
  └── 是否超预期：是（市场预期+30%）
     ↓
结论："中报预告利润+50%超预期 → 业绩印证逻辑 ✅"
```

### 6.4 输出

```
📋 业绩验证
  ├── 公司：长电科技（600584）
  ├── 事件：2026年中报业绩预告
  ├── 营收：+35% (超预期)
  ├── 利润：+50% (超预期)
  ├── 结论：业绩印证逻辑 ✅
  └── 📝 文字总结
```

### 6.5 与走势验证的关系

两条验证线**共同决定逻辑健康度**：

```
逻辑健康度
  ├── 📊 走势印证率（50%）
  │   └── 市场情绪层面
  └── 📋 业绩印证率（50%）
      └── 基本面层面

最佳：走势✅ + 业绩✅  → 逻辑健康
危险：走势✅ + 业绩❌  → 纯炒作，警惕
```

### 6.6 关联前置预判

业绩发布是**可预知的**。前置预判录入财报日期后，事件当天自动触发：
1. ✅ 走势验证（股价反应）
2. ✅ 业绩验证（财务数据）
全部自动完成。

## 8. 最强逻辑股池与报告

### 8.1 什么是最强逻辑股池

简放的三个股池体系：
| 股池 | 方式 | 用途 |
|:----|:----|:-----|
| 动量股池 | 系统全市场扫描 | 感知资金方向 |
| 新高股池 | 系统每日自动 | 判断市场强弱 |
| **最强逻辑股池** | **你分析逻辑后选股** | **核心持仓候选** |

我们的最强逻辑追踪系统，最终产出就是这个股池。

每日涨停扫描和新高扫描是支撑最强逻辑股池的两大自动验证引擎：

```
涨停扫描 → 发现逻辑信号 → 
                              → 验证已有逻辑 → 更新健康分 → 最强逻辑股池
新高扫描 → 确认逻辑定价 → 
                                            ↘ 发现新逻辑 → 推荐候选 → 手动确认
```

### 8.2 股池生成规则

```
聚焦逻辑 + 核心逻辑
  ↓
提取所有关联个股 → 去重
  ↓
为每只股票组装：
  ├── 所属逻辑（为什么选它）
  ├── 相关事件/研报列表
  ├── 走势验证结果（近30日涨跌幅）
  ├── 业绩验证结果（最新财报数据）
  └── 当前买点信号（如有）
```

## 13. 逻辑失效判断机制

```
系统自动检测

  ├── 板块排名持续下滑 → 提示
  ├── 最近3个事件印证率≤30% → 建议降级
  ├── 最新业绩不及预期 → 警示
  ├── 走势✅但业绩❌ → 警惕纯炒作
  └── 30天无新事件 → 提示归档

  └── 系统建议降级 → 可"忽略，保持活跃"
```

## 16. 推送通知机制

聚焦逻辑的异常提醒，支持三种推送方式：

| 方式 | 目标 | 说明 | 阶段 |
|:----|:-----|:-----|:----|
| 🔔 页面内报警池 | 盯盘页面报警区 | 复用现有报警系统，新增一条报警记录 | P1 |
| 💬 微信私信 | 你的微信 | 系统直接发送消息到你的微信 | P1 |
| 👥 微信群 | 微信群组 | 推送到注册的微信群 | P2 |

### 推送触发场景

| 触发条件 | 推送内容 |
|:---------|:---------|
| 聚焦逻辑关联个股集体走弱 | "先进封装加速关联个股连续3天跑输大盘" |
| 聚焦逻辑关联公司发布业绩预告 | "长电科技发布中报预告，利润+50%超预期" |
| 前置预判事件当天提醒 | "提醒：今天英伟达财报日" |
| 系统建议升降级 | "先进封装加速已累积4个事件，建议升级为聚焦" |

## 17. 设计决策汇总

| # | 问题 | 结论 |
|:-:|:----|:------|
| ① | 投喂方式 | 链接+PDF+抖音（后续） |

| ③ | 走势验证 | 全自动，聚焦逻辑主动推送 |
| ④ | L3自动抓新闻 | ❌ 暂不做 |
| ⑤ | 页面位置 | 独立页面 |
| ⑥ | 数据存储 | `data/logic_tracking.json` |
| ⑦ | 资料防丢 | 链接存快照，PDF存原文 |
| ⑧ | 确认环节 | 预览→编辑→保存 |
| ⑨ | 逻辑失效 | 系统警示 + 用户手动覆盖 |
| ⑩ | 搜索 | 标签筛选 + 关键词搜索 |
| ⑪ | 走势验证输出 | 个股+板块+文字总结 |
| ⑫ | 逻辑时间线 | 按时间展示全部事件 |
| ⑬ | **逻辑分层** | **聚焦1~3 / 核心 / 观察** |
| ⑭ | **前置预判** | **L1手动日历 + L2自动(未来)** |
| ⑮ | 系统打分 | 自动打分 + 人工修正覆盖 |
| ⑯ | **业绩验证** | **财报自动检测，对比逻辑预期，双验证定健康度** |
| ⑰ | **最强逻辑股池** | **自动从聚焦+核心逻辑提取个股，组装信息，一键报告** |
| ⑱ | **AI能力** | **后端调LLM API（DeepSeek等）做摘要、打标签、文字总结** |
| ⑲ | **推送通知** | **页面内报警池 + 微信私信 + 微信群（P2）** |
| ⑳ | **每日涨停扫描** | **每日收盘后自动拉取涨停板池，匹配逻辑标签。按连板数分三级打分，板块效应推送提醒** |
| ㉑ | **每日新高扫描** | **每日收盘后自动拉取新高股池，匹配逻辑标签。匹配成功健康分+3，匹配即推送** |
| ㉒ | **涨停/新高自动发现新逻辑** | **未匹配个股→LLM提炼共同特征→生成候选标签→预览待确认** |
| ㉓ | **逻辑关联图谱** | **上下游/同板块/竞争/互补关系。一个逻辑验证→关联逻辑加分** |
| ㉔ | **资金面辅助验证** | **板块资金净流入趋势作为第三维度。三验证定健康度（40%/40%/20%）** |
