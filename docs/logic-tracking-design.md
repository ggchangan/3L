## 1. 产品定位

这是一套**辅助你追踪市场最强逻辑**的工具。核心理念：

> **不是先想好逻辑再录入，而是从日常阅读的资料中自动提炼逻辑，系统帮你归类、跟踪、验证。**

| 你做什么 | 系统做什么 |
|:---------|:----------|
| 读公众号/研报/视频 → 发现逻辑方向 | 投喂资料后自动提取摘要、打标签、归类 |
| 判断逻辑是否还在、是否印证 | 自动展示关联个股走势、板块排名、买点信号 |
| 标记逻辑状态（活跃/失效） | 关联个股走弱时主动警示 |

## 2. 用户工作流

```
📥 信息输入
公众号(产业/复盘) | 研报(PDF) | 抖音视频 | 新闻链接
       ↓
⚙️ 系统自动处理
解析内容 → 提取摘要 → 识别行业/公司/核心逻辑
       ↓
👀 预览确认 ← 新环节
你可编辑摘要、调整行业/公司标签、修改逻辑归属
       ↓
💾 保存到 data/logic_tracking.json
       ↓
📊 走势验证（你手动触发）
展示关联个股走势 + 板块排名变化 + 文字总结
       ↓
🔄 定期审视
扫一眼逻辑列表 → 更新状态 → 淘汰失效逻辑
```

## 3. 核心概念

### 3.1 逻辑标签

逻辑标签是系统核心。你投喂资料时，系统自动提取核心逻辑，匹配已有标签或提示新建。

每条逻辑标签包含：
- **名称** — 如"先进封装加速"
- **关联行业** — 如 `[半导体, 封测]`
- **关联个股** — 如 `[600584, 002156]`
- **状态** — 活跃 / 验证中 / 淡化 / 失效（你手动标记优先）
- **统计** — 事件总数、最后一次事件时间、走势验证结果

### 3.2 资料条目

每条你投喂的资料构成一个条目：

```
📄 涛定律深度解读（公众号·飞跑的路）
  ├── 📝 摘要（自动提取）
  ├── 🏷️ 行业标签 → 关联板块走势     ← 自动 + 你可修改
  ├── 🏷️ 公司标签 → 关联个股买点信号   ← 自动 + 你可修改
  ├── 🏷️ 逻辑标签 → 归入某个逻辑看板   ← 自动推荐 / 你手动选 / 新建
  ├── 📎 原文链接 / PDF原文（防丢）
  ├── ⭐ 收藏标记（重要资料置顶）
  └── 📊 走势验证结果（你手动触发后产生）

其它元数据：投喂时间、来源类型（公众号/研报/抖音/新闻）
```

### 3.3 催化剂事件

每一条资料条目本身就是一个催化剂事件。多个事件按时间排列在逻辑标签下，构成**逻辑时间线**。

## 4. 功能模块

### 4.1 逻辑标签管理（P0）

| 功能 | 说明 |
|:----|:-----|
| 创建逻辑标签 | 输入名称 + 关联行业 + 关联个股 |
| 编辑标签 | 修改名称/行业/个股/状态 |
| 删除标签 | 删除后关联的资料不会消失，仅解除标签关联 |
| 标签列表 | 按状态筛选（活跃/验证中/淡化/失效）|
| 标签统计 | 每个标签显示事件数量 + 最新走势 + 验证结果 |

### 4.2 资料投喂与处理（P0 ~ P1）

| 功能 | 说明 |
|:----|:-----|
| 投喂链接 | 粘贴公众号/新闻链接，自动抓取正文 |
| 上传PDF | 上传研报PDF，自动解析文字+提取摘要 |
| 投喂抖音链接 | 粘贴抖音链接 → 调用提取程序 → 获取内容 |
| 自动提取摘要 | AI 提取核心内容摘要 |
| 自动打行业标签 | 识别文中涉及的行业/板块 |
| 自动打公司标签 | 识别文中涉及的A股公司 |
| 自动匹配逻辑标签 | 匹配已有逻辑，或推荐新建 |
| 预览确认 | **投喂后必须预览**，编辑标签后再保存 |

### 4.3 逻辑详情与时间线（P1）

| 功能 | 说明 |
|:----|:-----|
| 逻辑概览 | 关联行业/个股、整体走势、状态 |
| 时间线视图 | 按时间排列所有关联事件，每事件附走势验证结果 |
| 走势验证 | 手动触发，展示个股+板块+文字总结 |

### 4.4 搜索（P1）

| 功能 | 说明 |
|:----|:-----|
| 按逻辑标签筛选 | 选择标签，只看该标签下的资料 |
| 按关键词搜索 | 搜索摘要/全文内容 |

### 4.5 逻辑状态变化警示（P2）

系统主动检测以下情况并警示：
- 关联个股**连续N天跑输大盘**
- 关联板块**动量排名持续下滑**
- 逻辑标签**超过N天没有新事件**

你手动标记的状态（活跃/验证中/淡化/失效）**优先级高于系统判断**。

## 6. 数据模型

### 逻辑标签

```json
{
  "id": "tag-advanced-packaging",
  "name": "先进封装加速",
  "description": "涛定律推动先进封装技术变革，封测产业链进入高景气周期",
  "related_industries": ["半导体", "封测"],
  "related_stocks": ["600584", "002156", "002185", "300604"],
  "status": "active",
  "status_override": false,
  "event_count": 3,
  "last_event_at": "2026-05-27",
  "verify_score": null,
  "created_at": "2026-05-27",
  "updated_at": "2026-05-27"
}
```

| 字段 | 说明 |
|:----|:-----|
| `status` | active / watching / declining / invalid |
| `status_override` | true=用户手动标记，覆盖系统自动判断 |
| `verify_score` | confirmed / neutral / diverged / null（待验证） |
| `event_count` | 关联资料条目总数 |

### 资料条目

```json
{
  "id": "entry-001",
  "source_type": "wechat",
  "source_name": "飞跑的路",
  "title": "涛定律：重新定义先进封装",
  "summary": "华为提出涛定律...",
  "full_text": "全文快照，防止链接失效...",
  "url": "https://mp.weixin.qq.com/s/xxx",
  "pdf_path": null,
  "industries": ["半导体", "封测"],
  "companies": ["600584", "002156"],
  "logic_tags": ["tag-advanced-packaging"],
  "pinned": false,
  "fed_at": "2026-05-27 14:30",
  "verify": {
    "3d_return": 0.0,
    "5d_return": 0.0,
    "10d_return": 0.0,
    "sector_rank_before": 5,
    "sector_rank_after": 3,
    "buy_signal_count": 2,
    "summary": "事件后关联个股持续走强，板块排名上升，逻辑得到印证",
    "score": "confirmed",
    "verified_at": null
  }
}
```

| 字段 | 说明 |
|:----|:-----|
| `source_type` | wechat / pdf / douyin / news |
| `source_name` | 来源名（公众号名/研报机构/平台） |
| `summary` | AI 提取的摘要 |
| `full_text` | 链接类存全文快照；PDF类存提取文字 |
| `pdf_path` | PDF 原文文件路径 |
| `pinned` | 是否收藏置顶 |
| `verify.score` | confirmed / neutral / diverged |

### 存储文件

- 路径：`data/logic_tracking.json`
- 结构：`{ "tags": [...], "entries": [...], "updated_at": "..." }`

## 7. 信息源与处理流程

### 7.1 公众号文章 / 新闻链接

```
粘贴链接 → 系统抓取正文 → AI 提取摘要 → 打标签 → 展示预览 → 确认保存
```

保存内容：链接 + 摘要 + 全文快照（防链接失效）

### 7.2 研报 PDF

```
上传 PDF → 解析文字 → AI 提取摘要 → 识别行业/公司/核心逻辑 → 展示预览 → 确认保存
```

保存内容：PDF原文文件 + 提取的摘要 + 关键结论

### 7.3 抖音视频

```
粘贴抖音链接 → 调用提取程序 → 获取视频内容/文案 → 可编辑预览 → 确认保存
```

保存内容：链接 + 提取的文案摘要
> 抖音提取程序已有，后续部署到服务器

## 8. 走势验证

### 8.1 触发方式

你手动点击"走势验证"按钮触发，系统自动计算。

### 8.2 输出内容

| 内容 | 数据来源 |
|:----|:--------|
| **关联个股信息** | 复用 `get_stock_card()`，含个股走势+买点信号 |
| **板块排名变化** | 对比验证前后的板块动量排名 |
| **文字总结** | AI 根据数据生成一段话，如"事件后3天关联个股平均涨+5.2%，板块排名从#5升至#3，逻辑印证良好" |

### 8.3 打分

你根据展示的数据和文字总结，手动点选：
- ✅ 印证 — 逻辑得到市场认可
- ➖ 一般 — 走势不明显
- ❌ 背离 — 逻辑未被市场认可

## 10. 逻辑失效判断机制

双轨制：

```
系统自动检测（主动警示）
  ├── 关联个股连续跑输大盘 → 提示"建议审视"
  ├── 板块排名持续下滑 → 提示"关注"
  └── 超N天无新事件 → 提示"可能淡化"

用户手动标记（优先级更高）
  ├── 活跃 → 你标记为"已失效"
  └── 系统提示"建议审视" → 你选择"忽略，仍为活跃"
```

| 机制 | 说明 |
|:----|:-----|
| 系统 | 自动检测关联个股/板块异常，给出提示 |
| 用户 | 可随时手动修改逻辑状态，**优先级覆盖系统判断** |
| 标记策略 | 你标记失效后，系统不再对该逻辑发送警报 |

| **P1** | 搜索 | 按标签筛选 + 关键词搜索摘要/全文 |
| **P1** | 逻辑时间线 | 按时间排列所有事件，附走势验证结果 |
| **P2** | 走势验证打分 | 你手动点验/一般/背离，系统记录历史 |
