# 3L 交易决策助手 — 产品设计文档 v1

> 版本：v3.2 | 日期：2026-05-28 | 状态：草案

---

## 1. 产品定位

**一句话：** A股3L交易者的交易辅助系统，帮你把复盘结论变成可执行的计划，盯盘时按计划执行、偏差报警。

**核心价值：**
- 不算命、不荐股、不做决策
- 系统算数据、做报警
- 你判断、你做决策、你写计划

---

## 2. 用户画像

- 使用 简放 3L 交易体系（动量主线 / 最强个股 / 趋势跟随）
- 交易经验和纪律尚在积累中，需要系统辅助成长
- 每天做：收盘复盘 → 人工分析决策 → 写日志 → 第二天盯盘 → 再复盘

---

## 3. 产品核心理念

### 3.1 人机分工

| 谁 | 做什么 |
|:---|:---|
| **系统** | 数据采集、基础分析、信息呈现、偏差报警 |
| **你** | 判断分析结论、定仓位、选股票、写日志、执行复盘 |

### 3.2 人工决策边界（系统不参与）

以下决策全部由人工完成，系统只提供参考数据：

| 决策项 | 说明 |
|:---|:---|
| **趋势股筛选** | 人工看走势结构，判断是否适合走趋势策略 |
| **自选股/方向管理** | 决定跟踪哪些方向、扫描哪些板块、调出哪些 |
| **个股选择** | 系统出买点信号列表（可能30只），人工结合行业逻辑/产业链/研报筛选到3只以内 |
| **仓位分配** | 买多少、总仓位控多少，人工判断 |
| **买卖决策** | 条件触发后，人工决定是否执行 |

### 3.3 决策流水线

```
Day N（当天，工作台操作）
  ① 📊 复盘 review → 看结果
        │
        ▼
  ② 🧑 工作台（journal）
        个股分析 / 方向调整 / 趋势调整
        （但不定新计划，等明天复盘后再定）
        │
        ▼
  ③ 📝 保存日志（包含明日初步关注的标的）
        │
        ├─→ trade_journal_N.json（每日计划）
        └─→ alarms.json（报警独立持久化，不过期）
              ├─ 工作台保存时自动同步
              └─ 直到触发/禁用/删除才失效
        │
        ▼
Day N+1（盯盘当天）
  ④ 📡 monitor 加载计划 + 报警 → 盘中执行
        ├─ 今日计划（来自昨日+今日日志合并）
        ├─ 🔴 持仓止损（从持仓数据实时读，不存 alarms.json）
        │   └─ 每只持仓股自动展示止损价
        └─ 🟡 计划报警（来自 alarms.json，独立持久化）
            └─ 偏差报警 + 非持仓股的止损报警
        │
        ▼ 收盘后
  ⑤ 📊 复盘 review（Day N+1 的新数据）
        │
        ▼
  ⑥ 🧑 工作台（继续做 + 定新计划）
      ├─ 把昨天没做完的收尾
      ├─ 标记今日计划执行复盘
      └─ 根据Day N+1的复盘结果，制定Day N+2的新计划
```
### 3.4 四个核心模块

```
┌──────────────────────────────────────────────────────────┐
│  📚 知识库                ← 你积累，系统存               │
│  行业逻辑 / 产业链 / tips / 交易纪律 / 历史教训           │
│  （辅助你决策时参考）                                     │
├──────────────────────────────────────────────────────────┤
│  📊 复盘 review           ← 系统算，你看                 │
│  算数据 → 呈现                                             │
├──────────────────────────────────────────────────────────┤
│  🧑 工作台（journal）     ← 你做所有决策的地方            │
│  个股分析 + 自选调整 + 趋势调整 + 定计划 + 写日志          │
│  产出 = 交易日志（含明日计划）                              │
├──────────────────────────────────────────────────────────┤
│  📡 盯盘 monitor          ← 系统监测                     │
│  加载计划 → 三层观测 → 偏差报警                           │
└──────────────────────────────────────────────────────────┘
```

### 3.5 知识库 vs 交易技巧（tips）的分工

| | **知识库** | **交易技巧（tips）** |
|:---|:---|:---|
| 内容 | 产业链 / 研报 / 公司笔记 / 行业逻辑 | 交易规则 / 买点判断 / 卖点判断 |
| 用途 | 分析个股时引用 | 你学习→消化→优化系统 |
| 更新 | 不定时（看到研报就记） | 持续学习 |
| 独立页面 | ✓ | ✓ |

**知识库设计思路（待讨论）：** 每条知识笔记带标签系统，可关联：
- 所属行业/板块（如"电力"）
- 关联个股（如"德明利"）
- 关联的买点类型（如"回踩EMA5"）
- 自由关键词

在个股分析/方向选择时，系统自动匹配命中标签，在侧边栏展示"相关知识"。
也可通过搜索个股名/产业名直接查找相关笔记。

### 3.6 非交易系统页面（项目管理类）

以下页面与交易系统无关，属于项目管理工具：

- **skills.html** — 记录与 Hermes 交互产生的技能，追踪技能变化
- **每日成果看板** — 追踪产出成果、重要文档、待办事项

**处理方案：** 移出主导航，路径隔离，仅在底部小字入口可见。保留在同服务器，不拆分为独立项目。

### 3.7 外围数据接入

**范围：** 美股重点个股涨跌 + 商品期货

**数据来源：** 系统自动获取行情数据

**映射关系（用户提供）：** 每条美股/商品数据需关联到影响的A股板块

| 用户提供示例 | 说明 |
|:---|:---|
| 英伟达 → 影响算力/AI链 | 英伟达股价波动影响A股算力板块 |
| 苹果 → 影响消费电子链 | 苹果股价影响A股果链 |
| 原油 → 影响能源/化工 | 原油期货影响A股化工板块 |
| 铜 → 影响有色/电力设备 | 铜期货影响A股有色板块 |

**用户需要提供的：** 美股个股清单（代码+名称）+ 对应的A股影响映射。格式待定，用户准备好后提供。

**展示位置：** monitor 计划层和信息层之间，开盘前突出显示，盘中可收起。

### 3.8 导航排序

```
📡 盯盘  |  📊 复盘  |  🧑 工作台  |  自选  |  个股  |  趋势  |  行业  |  涨幅  |  宏观  |  tips
 ╰─────── 三大核心（日流程）────────╯  ╰─────────── 独立工具（随时用）────────────╯
                                                                                
底部小字：⚙ 项目管理  |  📋 成果看板
```

说明：
- 原 journal 页面被工作台取代，导航中不保留
- tips 保持现有位置不动
- **持仓页面（private/holdings.html）** 为独立页面，受 Basic Auth 保护，不公开显示在导航栏
- 首页（/）默认跳转到盯盘页面（monitor.html），index.html 文件保留
- skills/成果看板移出主导航，路径隔离，走底部入口

### 3.9 盯盘三层模型

| 层级 | 内容 | 报警类型 |
|:---|:---|:---|
| ① **规则层** | 交易纪律（自己最易犯的错误置顶） | 无（视觉提醒） |
| ② **计划层** | 今日计划（来自日志的买/卖/观察）+ 🔴持仓止损 + 🟡计划报警 | 条件触发/偏差报警 |
| ③ **信息层** | 大盘/主线/个股实时数据 | 异常波动/预期偏差 |

#### 报警类型

| 报警类型 | 触发条件 | 优先级 | 声音 |
|:---|:---|:---:|:---:|
| 🟢 **机会到** | 计划中的买入条件满足 | 高 | 一声"叮" |
| 🔴 **止损警** | 计划中的卖出条件满足 | 最高 | 急促"叮叮叮" |
| 🟡 **预期偏差** | 大盘走势与昨日判断不符 | 中 | 一声"咚" |
| 🔔 **异常提醒** | 突发板块拉升/外围变化 | 中 | 短促"嘟嘟" |
| ⏰ **收盘提醒** | 收盘前计划条件尚未触发 | 低 | 轻"叮" |

#### 报警交互设计

- **轮询检测：** monitor 每30秒前端轮询，比对前后数据状态变化，发现新报警时触发
- **声音播报：** 所有报警类型均带声音，使用 Web Audio API 生成（无需额外音频文件）
- **文字更新：** 报警层区域实时更新文字说明
- **浏览器通知：** 切换页面时，屏幕右上角弹出系统通知

## 6. 🌟 龙头观测重构方案（v3.2）

> 2026-05-28 | 状态：设计中（待实现）

### 6.1 现状痛点

当前龙头观测区块显示**155个行业、269只龙头**的全量列表，存在三个问题：

| 问题 | 描述 |
|:---|:---|
| **信息过载** | 16页翻不完，关注的行业淹没在大量无关数据中 |
| **静态展示** | 只显示涨跌幅/市值，缺少动态变化信息 |
| **价值不足** | 看完没有 actionable 的结论 |

### 6.2 核心思路（源自 简放 第5期训练营"如何盯盘"）

行业龙头盯盘的 4 个关键方向：

| 方向 | 意义 | 盯什么 |
|:---|:---|:---|
| ① **龙头异动** | 龙头带头，板块跟风 | 放量突破 / 领涨领跌 / 新高 |
| ② **龙头切换** | 板块内部结构变化 | #2/#3 超越 #1（市值龙头） |
| ③ **龙头 vs 板块背离** | 板块持续性判断 | 龙头涨跌 vs 板块指数涨跌差异 |
| ④ **市场龙头梯队联动** | 情绪周期 / 主线确认 | 连板高度 / 最高板倒了 vs 主线重合度 |

## 7. 设计原则

### 7.1 人机职责清晰
- 系统不算"应该买什么"，只提供数据和分析
- 你永远是决策者
- 计划必须人工写入，系统不代劳

### 7.2 人工决策边界明确
- 趋势股筛选、自选管理、个股选择、仓位分配、买卖决策 → 全部人工
- 系统提供数据参考，不替你做判断

### 7.3 信息分层
- 复盘从上到下 = 阅读顺序
- 工作台从待办到产出 = 工作流顺序
- 盯盘从上到下 = 优先级顺序
- 不堆砌信息，每个区块有明确目的

### 7.4 闭环可追踪
- 每个计划都有状态（pending/triggered/executed/not_triggered/missed）
- 每个操作都有追溯："当时为什么决定做"
- 每笔交易都能复盘："计划执行了没有"

### 7.5 辅助成长
- 纪律置顶 → 行为矫正
- 交易反思 → 经验积累
- 执行复盘 → 能力提升
