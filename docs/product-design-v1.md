# 3L 交易决策助手 — 产品设计文档 v1

> 版本：v1.1 | 日期：2026-05-25 | 状态：草案

---

## 1. 产品定位

**一句话：** A股3L交易者的交易辅助系统，帮你把复盘结论变成可执行的计划，盯盘时按计划执行、偏差报警。

**核心价值：**
- 不算命、不荐股、不做决策
- 系统算数据、做报警
- 你判断、你做决策、你写计划

---

## 2. 用户画像

- 使用 简放 3L 交易体系（动量主线 / 最强个股 / 趋势跟随）
- 交易经验和纪律尚在积累中，需要系统辅助成长
- 每天做：收盘复盘 → 人工分析决策 → 写日志 → 第二天盯盘 → 再复盘

---

## 3. 产品核心理念

### 3.1 人机分工

| 谁 | 做什么 |
|:---|:---|
| **系统** | 数据采集、基础分析、信息呈现、偏差报警 |
| **你** | 判断分析结论、定仓位、选股票、写日志、执行复盘 |

### 3.2 人工决策边界（系统不参与）

以下决策全部由人工完成，系统只提供参考数据：

| 决策项 | 说明 |
|:---|:---|
| **趋势股筛选** | 人工看走势结构，判断是否适合走趋势策略 |
| **自选股/方向管理** | 决定跟踪哪些方向、扫描哪些板块、调出哪些 |
| **个股选择** | 系统出买点信号列表（可能30只），人工结合行业逻辑/产业链/研报筛选到3只以内 |
| **仓位分配** | 买多少、总仓位控多少，人工判断 |
| **买卖决策** | 条件触发后，人工决定是否执行 |

### 3.3 决策流水线

```
📊 复盘 review
  系统产出：大盘结论 / 主线排名 / 持仓回顾 / 买点信号列表（20-30只）
      │
      ▼
🧑 工作台（交易日志页面改造）
  ┌────────────────────────────────────────────┐
  │  ① 个股分析                                │
  │    → 从信号列表点开候选股                   │
  │    → 看技术面 + 财务面 + 消息面             │
  │    → 判断"这只做不做"                       │
  │                                             │
  │  ② 自选股/方向调整                          │
  │    → 主线变了，方向要不要调                  │
  │    → 新增/删除自选股                        │
  │                                             │
  │  ③ 趋势股调整                              │
  │    → 哪些股适合趋势策略                      │
  │    → 加/减趋势跟踪列表                      │
  │                                             │
  │  ④ 定计划（综合①②③）                       │
  │    → 买入：德明利 | 200股 | 条件：回踩+缩量  │
  │    → 卖出：宁德   | 全部 | 条件：跌破250     │
  │    → 观察：电力板块 | 是否加速分化           │
  │                                             │
  │  ⑤ 写今日操作/反思（自由文本）               │
  │  ─────────────────────                      │
  │  [💾 保存 → 生成交易日志]                    │
  └────────────────────────────────────────────┘
      │
      ▼
📡 盯盘 monitor
  加载计划 → 三层观测 → 偏差报警
      │
      ▼ 第二天收盘
📝 交易日志标记执行结果
```

### 3.4 四个核心模块

```
┌──────────────────────────────────────────────────────────┐
│  📚 知识库                ← 你积累，系统存               │
│  行业逻辑 / 产业链 / tips / 交易纪律 / 历史教训           │
│  （辅助你决策时参考）                                     │
├──────────────────────────────────────────────────────────┤
│  📊 复盘 review           ← 系统算，你看                 │
│  算数据 → 呈现                                             │
├──────────────────────────────────────────────────────────┤
│  🧑 工作台（journal）     ← 你做所有决策的地方            │
│  个股分析 + 自选调整 + 趋势调整 + 定计划 + 写日志          │
│  产出 = 交易日志（含明日计划）                              │
├──────────────────────────────────────────────────────────┤
│  📡 盯盘 monitor          ← 系统监测                     │
│  加载计划 → 三层观测 → 偏差报警                           │
└──────────────────────────────────────────────────────────┘
```

### 3.5 知识库 vs 交易技巧（tips）的分工

| | **知识库** | **交易技巧（tips）** |
|:---|:---|:---|
| 内容 | 产业链 / 研报 / 公司笔记 / 行业逻辑 | 交易规则 / 买点判断 / 卖点判断 |
| 用途 | 分析个股时引用 | 你学习→消化→优化系统 |
| 更新 | 不定时（看到研报就记） | 持续学习 |
| 独立页面 | ✓ | ✓ |

**知识库设计思路（待讨论）：** 每条知识笔记带标签系统，可关联：
- 所属行业/板块（如"电力"）
- 关联个股（如"德明利"）
- 关联的买点类型（如"回踩EMA5"）
- 自由关键词

在个股分析/方向选择时，系统自动匹配命中标签，在侧边栏展示"相关知识"。
也可通过搜索个股名/产业名直接查找相关笔记。

### 3.6 非交易系统页面（项目管理类）

以下页面与交易系统无关，属于项目管理工具：

- **skills.html** — 记录与 Hermes 交互产生的技能，追踪技能变化
- **每日成果看板** — 追踪产出成果、重要文档、待办事项

**处理方案：** 移出主导航，路径隔离，仅在底部小字入口可见。保留在同服务器，不拆分为独立项目。

### 3.7 导航排序

```
📡 盯盘  |  📊 复盘  |  🧑 工作台  |  自选  |  个股  |  趋势  |  行业  |  涨幅  |  宏观  |  tips
 ╰─────── 三大核心（日流程）────────╯  ╰─────────── 独立工具（随时用）────────────╯
                                                                                
底部小字：⚙ 项目管理  |  📋 成果看板
```

### 3.8 盯盘三层模型

| 层级 | 内容 | 报警类型 |
|:---|:---|:---|
| ① **规则层** | 交易纪律（自己最易犯的错误置顶） | 无（视觉提醒） |
| ② **计划层** | 今日计划（来自日志的买/卖/观察） | 条件触发/偏差报警 |
| ③ **信息层** | 大盘/主线/个股实时数据 | 异常波动/预期偏差 |

#### 报警类型

| 报警类型 | 触发条件 | 优先级 | 声音 |
|:---|:---|:---:|:---:|
| 🟢 **机会到** | 计划中的买入条件满足 | 高 | 一声"叮" |
| 🔴 **止损警** | 计划中的卖出条件满足 | 最高 | 急促"叮叮叮" |
| 🟡 **预期偏差** | 大盘走势与昨日判断不符 | 中 | 一声"咚" |
| 🔔 **异常提醒** | 突发板块拉升/外围变化 | 中 | 短促"嘟嘟" |
| ⏰ **收盘提醒** | 收盘前计划条件尚未触发 | 低 | 轻"叮" |

#### 报警交互设计

- **轮询检测：** monitor 每30秒前端轮询，比对前后数据状态变化，发现新报警时触发
- **声音播报：** 所有报警类型均带声音，使用 Web Audio API 生成（无需额外音频文件）
- **文字更新：** 报警层区域实时更新文字说明
- **浏览器通知：** 切换页面时，屏幕右上角弹出系统通知

## 5. 数据流

```
review（系统产出）
  ├─ 大盘结论（结构/阶段/成交额评价）
  ├─ 主线排名（动量排名/变化趋势）
  ├─ 持仓回顾（持仓状态/止损距离）
  └─ 买点信号列表（20-30只，你标记3只重点）

       ↓ 你带着候选股去工作台

工作台（journal页面改造）
  ├─ 个股分析（技术+财务+消息，决定做不做）
  ├─ 自选股/方向调整
  ├─ 趋势股调整
  ├─ 产出：明日计划（buy/sell/watch + 条件+仓位）
  └─ 产出：今日操作/反思

       ↓ 保存 = 写入交易日志

monitor（加载计划）
  ├─ 计划层：显示今日计划
  ├─ 信息层：大盘/板块/个股实时数据
  └─ 报警层：计划条件 vs 实时行情 → 偏差报警

       ↓ T+1日

工作台标记执行结果
```

## 6. 设计原则

### 6.1 人机职责清晰
- 系统不算"应该买什么"，只提供数据和分析
- 你永远是决策者
- 计划必须人工写入，系统不代劳

### 6.2 人工决策边界明确
- 趋势股筛选、自选管理、个股选择、仓位分配、买卖决策 → 全部人工
- 系统提供数据参考，不替你做判断

### 6.3 信息分层
- 复盘从上到下 = 阅读顺序
- 工作台从待办到产出 = 工作流顺序
- 盯盘从上到下 = 优先级顺序
- 不堆砌信息，每个区块有明确目的

### 6.4 闭环可追踪
- 每个计划都有状态（pending/triggered/executed/not_triggered/missed）
- 每个操作都有追溯："当时为什么决定做"
- 每笔交易都能复盘："计划执行了没有"

### 6.5 辅助成长
- 纪律置顶 → 行为矫正
- 交易反思 → 经验积累
- 执行复盘 → 能力提升
