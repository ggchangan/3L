
## 1. 背景与问题

**现状：** 个股分析页（StockAnalysis）已有诊断、StockCard、回测，但缺少财报基本面数据。用户分析个股时需要跳转到同花顺/东财查看财务指标，不方便。

**痛点：** 
- 技术面（K线/买点/信号）和基本面（营收/利润/ROE）分离在两个系统
- 想看财务数据要手动复制代码去同花顺搜

**目标：** 在个股分析页内直接展示近2年财务报告摘要，每期关键指标一目了然。

## 2. 设计思想

### 2.1 核心理念

财报速览 = 快速回答"基本面好不好"，不是取代同花顺图解财报的详细图表，而是给技术分析配一个基本面对照。

### 2.2 方案选择

| 候选方案 | 优点 | 缺点 | 结论 |
|:--------|:----|:----|:----:|
| 嵌入同花顺图解财报iframe | 现成内容，功能完整 | 依赖外站、加载慢 | ❌ |
| 自建财报卡（新浪三表API） | 数据自有可控，与系统集成好 | 需开发后端+前端，字段有限 | ✅ |
| 外链跳转同花顺 | 零开发 | 跳出系统体验差 | ❌ |

### 2.3 设计原则

- **简洁优先：** 只展示6个核心指标（营收/净利/毛利率/净利率/ROE/EPS），不堆砌
- **对比直观：** 每个指标同时显示绝对值 + 同比变化（颜色标识）
- **加载流畅：** 后端缓存财务数据，不每次实时拉取

### 2.4 屏幕内外

- **做什么：** 个股分析页展示近2年（8个报告期）的核心财务指标
- **不做什么：** 不做跨股对比、不做财务图表（折线图/柱状图）、不做估值计算

## 3. 数据模型

### 3.1 API响应格式

```json
GET /api/stock-financial-reports?code=600584

{
  "success": true,
  "name": "长电科技",
  "code": "600584",
  "reports": [
    {
      "period": "2026-03-31",
      "label": "2026一季报",
      "indicators": {
        "revenue": {"value": 129.3, "unit": "亿", "yoy": 12.5},
        "net_profit": {"value": 18.7, "unit": "亿", "yoy": 8.2},
        "gross_margin": {"value": 32.4, "unit": "%", "yoy": -1.2},
        "net_margin": {"value": 14.5, "unit": "%", "yoy": null},
        "roe": {"value": 15.6, "unit": "%", "yoy": null},
        "eps": {"value": 1.23, "unit": "元", "yoy": null}
      }
    }
  ],
  "periods": ["2026-03-31", "2025-12-31", "2025-09-30", ...],
  "source": "sina_finance"
}
```

### 3.2 数据流

```
新浪财报三表API (sina_financial_report)
  → 后端解析利润表/资产负债表
  → 提取关键指标，缓存到本地JSON（每日刷新）
  → 前端通过 REST API 获取
  → StockAnalysis 页面展示折叠卡片
```

## 4. 系统设计

### 4.1 API设计

**`GET /api/stock-financial-reports?code=600584`**

| 参数 | 类型 | 必填 | 说明 |
|:----|:----|:----|:----|
| code | string | 是 | 6位股票代码 |

响应：见3.1节Schema。

### 4.2 前端设计

**位置：** StockAnalysis页，诊断区块和回测之间

**交互：**
- 默认折叠，显示"📋 财报速览 ▶"
- 展开后展示报告期Tab（最多8期，近2年）
- 默认选中最新一期，显示6个指标卡片
- 点击Tab切换显示对应期数据
- 负值增长用红色，正值用绿色

**组件：**
- `FinancialReports.tsx` — 财报速览组件
- 嵌入到 `StockAnalysis.tsx` 的render中

### 4.3 后端设计

- `server/backend/api/stock_financial.py` — API handler
- `server/backend/services/financial_report_service.py` — 数据获取+缓存逻辑
- 数据源：新浪财报三表API（`quotes.sina.cn`）

## 6. 附录

### 6.1 数据源说明

使用新浪财报三表API（`sina_financial_report`），利润表（lrb）可获取：
- 营业收入、营业成本、毛利率（计算）、净利润、每股收益

资产负债表（fzb）可获取：
- 净资产（用于ROE计算）
