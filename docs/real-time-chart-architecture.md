# 实时K线图架构设计 v1

> 版本：v1.0 | 日期：2026-05-25 | 状态：草案

---

## 1. 问题

所有K线图（大盘图/板块图/个股图）数据只到上一交易日，盘中不更新。根源是日K线数据源只在收盘后生成，盘中不存在"今天的日K线"。

**影响范围：**
- 盯盘页面：板块K线图、个股K线图
- 复盘页面：个股诊断图
- 趋势候选页面：趋势K线图

**风险：** 盘中用户点开图表永远看到昨天数据，导致误判。

---

## 2. 设计原则

1. **按需生成，不预生成** — 用户点📊才生成SVG，不浪费扫描算力
2. **数据复用** — 同一股票的60日K线JSON只缓存一份，多个图共享
3. **盘中实时叠加** — 日K线主体 + 今日实时行情虚线标注
4. **新鲜度可见** — 每张图底部标注「数据截至」时间戳
5. **统一端点** — 所有图表走同一个 `/api/stock-chart` 端点

---

## 3. 数据流

```
用户点📊 → /api/stock-chart?code=XXXX
                   │
            ┌──────┴──────┐
            │  data_layer │ 检查60日K线缓存
            │  .get_60d() │ (缓存1h，过期自动拉)
            └──────┬──────┘
                   │ 60根日K线数据（YYYYMMDD格式）
            ┌──────┴──────┐
            │  腾讯实时    │ 拉1次实时行情
            │  .get_qt()  │ (今日开/高/低/现价/成交量)
            └──────┬──────┘
                   │ 今日实时数据
            ┌──────┴──────┐
            │  SVGGEN     │ 合成SVG
            │  .render()  │ 60根实线 + 1根虚线
            └──────┬──────┘
                   │ SVG字符串 → 返前端
```

### 3.1 缓存策略

| 缓存 | TTL | 说明 |
|:----|:---|:-----|
| 60日K线JSON | 1小时 | 过期自动用akshare/腾讯重拉 |
| 腾讯实时行情 | 不缓存 | 每次调用实时拉取 |
| 生成后的SVG | 交易时间10分钟 | 用文件修改时间判断，非交易时间1h |

## 4. SVG渲染规则

### 4.1 日K线主体（60根实线蜡烛）

- 取60日收盘价/开盘价/最高/最低
- 阳线：红色实心（当日涨）
- 阴线：绿色实心（当日跌）
- 均线：EMA5(黄)、EMA10(蓝)、EMA20(紫)

### 4.2 今日实时虚线蜡烛

- 在60根蜡烛右侧增加第61根
- **绘制规则：**
  - 开盘价 = 腾讯实时行情中的今日开盘价
  - 最高价 = 腾讯实时行情中的今日最高价（盘中动态变化）
  - 最低价 = 腾讯实时行情中的今日最低价（盘中动态变化）
  - 收盘价 = 腾讯实时行情中的当前价（盘中不断更新）
- **视觉风格：**
  - 蜡烛实为**半透明虚线框**（`stroke-dasharray="4,3"`，`opacity="0.6"`）
  - 开盘和当前价之间用虚线矩形表示
  - 最高/最低用虚线影线延伸到虚线框外
  - 若当前价 > 开盘价，虚线框红色；当前价 < 开盘价，绿色
- **浮标标注：** 在虚线框右侧标注 `今开/今高/今低/现价`

### 4.3 今日成交量虚线柱

- 在成交量区域增加第61根虚线柱
- **预估算法：** `预估值 = 当前成交量(手) × (240分钟 ÷ 已过分钟数)`
  - 开盘(09:30)起算，到当前时间的分钟数
  - 如14:00：已过270分钟，预估值 = 当前量 × 240/270
- **视觉：** 半透明虚线柱，顶部标注「预」和数值
- 上午收盘(11:30)前后递进修正

### 4.4 数据新鲜度标注

底部标注：
```
数据截至: 2026-05-25 14:35
历史日K线: 2026-05-22 15:00 (上一交易日收盘)
今日实时: 🟢 盘中更新
```

收盘后（15:00后）：
```
数据截至: 2026-05-25 15:00 (已收盘)
```

**请求：** `GET /api/stock-chart?code=688428&t=1234567890`

|:----|:----|:-----|

## 6. 前后端改动清单

### 后端新增 `/api/stock-chart`

文件：`backend/api/stock.py`（新增 `_handle_stock_chart`）

```python
def _handle_stock_chart(h, path):
    code = parse_query(path).get('code', [None])[0]
    if not code:
        h.send_json({'error': 'missing code param'})
        return
    svg_str, err = generate_stock_chart(code)
    if err:
        h.send_json({'error': err})
        return
    h.send_response(200, svg_str.encode(), 'image/svg+xml')
```

### 核心函数 `generate_stock_chart(code)`

1. `data_layer.get_60d_klines(code)` — 获取60日K线（含缓存逻辑）
2. `get_realtime_kline(code)` — 获取腾讯实时行情
3. 合成SVG渲染逻辑

### 前端改动

**`monitor.js` — `toggleMChart(id, code)`：**
```javascript
// 旧：用静态SVG文件
obj.data = '/pub/charts/' + code + '.svg?t=' + Date.now();

// 新：用按需API生成
obj.data = '/api/stock-chart?code=' + code + '&t=' + Date.now();
```

**扫描脚本 `scan_buy_signals.py`：**
- 不再预生成SVG（省算力）
- 信号dict增加 `'signal': 'buy'` 字段

### 复盘页面

`toggleChart(id, code)` 同样改为调 `/api/stock-chart`。

### 中证全指图

`gen_index_chart.py` 输出时叠加腾讯实时行情标注。

## 7. 板块5日涨幅修正

**问题：** 当前仅从今日涨幅TOP15里算5日涨幅，漏了很多5日涨得好的板块。

**修复：** 
1. 获取全部90个行业的今日涨跌幅
2. 从缓存的60日K线中读取每个板块的5日涨幅
3. 对全部板块按5日涨幅排序取TOP10
4. 缓存24小时（60日K线中的数据不频繁变化）

**注意：** 首次计算全部90个板块需要确保60日K线缓存已就绪（需要跑过一轮板块K线拉取）。

## 8. 边界情况

| 场景 | 处理 |
|:----|:-----|
| 非交易时间（盘前/盘后/周末） | 不拉实时行情，只显示日K线+标注「已收盘」 |
| 实时行情API失败 | 只显示日K线，底部标注「实时行情获取失败」 |
| 60日K线缓存过期且拉取失败 | 显示缓存中的旧K线+标注「数据可能过旧」 |
| 刚开盘（09:30-09:35） | 实时行情已有但量很少，虚线框正常显示，成交量预估可能不准 |
| 股票停牌 | 今日实时行情无数据，只显示日K线，标注「今日停牌」 |
