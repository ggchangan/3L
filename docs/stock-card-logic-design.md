## 第1章 大盘周期判定（波峰波谷）

### 概述

基于中证全指（000985）K线数据，使用V5置信度评分法判断大盘处于波峰/波谷还是波中位置，直接映射到总仓位水位。

### 数据源

- 指数：中证全指（000985，腾讯财经API，sh前缀）
- 数据频率：日K线，含 open/close/high/low

### 算法：V5 四条件评分法

4个条件各自独立0/1判定，累计评分 pk_score / vl_score（0~4分）：

#### 波峰判定（pk_score）

| 条件编号 | 条件 | 说明 |
|:--------|:-----|:-----|
| pk1 | bias20 < -2%（5日前） | 5日前乖离率<-2%，不可能是极端的波峰（排除刚暴跌的误判） |
| pk2 | bias20峰值 <-2%（20日内） | 20日内的峰值还< -2%，说明从未到过热区域 |
| pk3 | 5日峰谷因子 ≤1 | 近期价格未创新高 |
| pk4 | bias5 > 0（20日峰值偏正） | 近5日偏离偏正，有冲高动力 |

偏波峰：pk_score ≥ 4 或 bias20 > 8%

#### 波谷判定（vl_score）

| 条件编号 | 条件 | 说明 |
|:--------|:-----|:-----|
| vl1 | bias20 > 2%（5日前） | 5日前乖离率>2%，不可能是极端波谷 |
| vl2 | bias20低谷 <2%（20日内） | 20日内的低谷还<2%，说明从未到过冷区域 |
| vl3 | 5日峰谷因子 ≥ -1 | 近期价格未创新低 |
| vl4 | bias5 < 0（20日低谷偏负） | 近5日偏离偏负，有下行惯性 |

偏波谷：vl_score ≥ 4 或 bias20 < -8%

#### 仓位映射

| 档位 | 判定条件 | 仓位策略 |
|:----|:---------|:--------|
| 偏波峰 | pk ≥ 4 或 bias20 > 8% | 五成仓位，控仓 |
| 波中偏上 | pk ≥ 3 | 六至七成，注意减仓 |
| 波中 | pk < 3 且 vl < 3 | 七至八成，正常交易 |
| 波中偏下 | vl ≥ 3 | 五至七成，收紧止损 |
| 偏波谷 | vl ≥ 4 或 bias20 < -8% | 五至八成，积极找买点 |

### 核心原则

不要检测"现在有多高/多低"，要检测"是不是在掉头"。
bias20 = (close - EMA20) / EMA20 × 100

## 第2章 主线判定

### 概述

基于同花顺90个行业板块20日涨幅排序，判定哪些板块属于主线（最强动量方向）。

### 数据源

- `akshare.stock_board_industry_index_ths()` — 同花顺行业板块指数
- 需显式传 start/end 日期参数

### 算法

```
  对90个行业板块按20日涨幅降序排列
  ┌────────────────┐
  │  前5名 → 主线   │  第一仓位优先
  ├────────────────┤
  │  6~10名 → 次级  │  活跃但不够强
  ├────────────────┤
  │  其余 → 非主线   │  不在资金共识方向
  └────────────────┘
```

### 得分计算

每个板块有 momentum_score（动量得分），基于多因素评分（非简单排名）。

### 主线和"方向"的关系

- 行业板块（同花顺90个）≠ 用户自定义的"方向"（如"商业航天"）
- 通过 `stock_industry_map.json` 的 `ths_industry` 字段映射
- 个股的 sector 判断：`stock_info.get('ths_industry', '')`

        ┌────────┼────────┐

   │ 前5名  │ │6-10名│ │其余   │

## 第3章 结构判定

### 概述

判断个股处于什么结构（上涨趋势/下降趋势/区间震荡）。这是整个卡片计算的核心第一步，后续所有买点/卖点/阶段/信号都依赖它。

**代码位置：** `backend/core/ema_utils.py` → `get_structure(closes)`

### 算法：EMA12不对称斜率阈值法

#### 基本原理

基于EMA12的线性回归斜率 + BIAS乖离率，使用**不对称阈值**：

| 方向 | 条件 | 阈值设计意图 |
|:----|:-----|:------------|
| **上涨趋势** | EMA12斜率 > 0.8% + BIAS > -5% | **严格** — 只认斜率陡+价格在EMA12附近的强势上涨 |
| **下降趋势** | EMA12斜率 < -0.2% + BIAS < 3% | **宽松** — 只要斜率稍弱+价格不严重超卖就判下降 |
| **区间震荡** | 两者都不满足 | — |

#### 为什么用不对称阈值？

回测发现自选股数据天然偏多（上涨样本占70%+），用对称阈值会导致：
- 上涨判得太多（73%+），很多震荡被误判为上涨
- 下降判得太少，且判了下降后还是涨的（回测+1.00%）

不对称阈值将上涨门槛提高，下降门槛降低，结果更平衡：

| 指标 | 旧版(EMA20对称) | 新版(EMA12不对称) | 改善 |
|:----|:--------------:|:----------------:|:----:|
| 上涨占比 | 73.9% | **51.6%** | ↓22% |
| 上涨5日均收益 | +4.45% | **+5.12%** | ↑0.67% |
| 下降占比 | 6.0% | **6.1%** | 略增 |
| 下降5日均收益 | +1.00% | **+0.24%** | ↓0.76% |
| 区间震荡占比 | 20.0% | **42.3%** | ↑22% |
| **区分度**(涨减跌) | +3.45% | **+4.88%** | ↑1.43% |

#### 算法细节

```python
def get_structure(closes):
    if len(closes) < 25:
        return '--'
    
    ema12 = ema_list(closes, 12)
    e12_recent = [v for v in ema12[-12:] if v is not None]
    if len(e12_recent) < 5:
        return '--'
    
    slope = _reg_slope(e12_recent)         # 线性回归斜率
    slope_pct = slope / e12_recent[0] * 100 if e12_recent[0] else 0
    
    cur, cur_ema12 = closes[-1], e12_recent[-1]
    bias = (cur - cur_ema12) / cur_ema12 * 100 if cur_ema12 else 0
    
    if slope_pct > 0.8 and bias > -5:
        return '上涨趋势'
    elif slope_pct < -0.2 and bias < 3:
        return '下降趋势'
    else:
        return '区间震荡'
```

#### 参数含义

| 参数 | 值 | 说明 |
|:----|:--|:-----|
| EMA周期 | 12 | 比EMA20更灵敏，比EMA10更稳定 |
| 回看窗口 | 12根 | 斜率计算的K线数量 |
| 上涨斜率阈值 | >0.8% | 斜率超过0.8%/日才算上涨 |
| 上涨乖离阈值 | >-5% | 价格不能低于EMA12的5%以上 |
| 下降斜率阈值 | <-0.2% | 斜率只要低于-0.2%/日就算下降 |
| 下降乖离阈值 | <3% | 价格不能高于EMA12的3%以上 |

#### 参数优化过程

通过三阶段扫描确定最优参数：

1. **粗扫**(500种组合，EMA10/15/20/30)：锁定EMA10系列最优
2. **细扫**(400种组合，EMA10窗口10-15)：TOP1 → EMA10/窗11/斜率0.55/乖离4.0
3. **深度扫描**(1240种组合，不对称阈值+排列过滤+多因子评分)：最优 → **EMA12不对称阈值**

三种方案对比：

| 方案 | 区分度 | 上涨% | 上涨收益 | 下降收益 | 震荡% |
|:----|:-----:|:-----:|:-------:|:-------:|:----:|
| EMA20对称阈值(旧) | +3.45% | 73.9% | +4.45% | +1.00% | 20.0% |
| EMA10对称阈值(细扫) | +4.73% | 63.4% | +4.71% | -0.02% | 33.1% |
| **EMA12不对称阈值** | **+4.88%** | **51.6%** | **+5.12%** | **+0.24%** | **42.3%** |

## 第4章 阶段判定

### 概述

在结构确定后，进一步判断当前处于什么阶段（加速/上行/滞涨等），基于EMA10斜率比的精细分析。

**代码位置：** `backend/core/ema_utils.py` → `get_stage(closes, structure, highs, lows, volumes)`

### 两类分支

阶段判定分两大路径：

#### 路径A：区间震荡 → 价格位置法

```python
if structure == '区间震荡':
    cur = closes[-1]
    lo = support_level or min(lows[-15:])   # 15日最低
    hi = resistance_level or max(highs[-15:])  # 15日最高
    if hi > lo:
        pct = (cur - lo) / (hi - lo) * 100
        if pct < 30:    return '区间底部'
        elif pct > 70:  return '区间顶部'
        else:           return '区间中段'
    return '--'
```

| 位置 | 阈值 | 含义 |
|:----|:----|:-----|
| 区间底部 | pct < 30% | 靠近支撑，考虑加仓或买入 |
| 区间中段 | 30% ≤ pct ≤ 70% | 方向不明，等待边界 |
| 区间顶部 | pct > 70% | 靠近压力，考虑减仓或卖出 |

#### 路径B：上涨/下降趋势 → EMA10斜率比法

```python
e10 = ema_list(closes, 10)
e10_last = [v for v in e10[-15:] if v is not None]
s1 = _reg_slope(e10_last)       # 末15根整体斜率（趋势基准）
s2 = _reg_slope(e10_last[-3:])  # 末3根斜率（近期动量）
```

##### 上涨趋势阶段判定

| s1 | s2 | 条件 | 阶段 | 含义 | 定位 |
|:--|:---|:-----|:----|:-----|:----|
| >0 | >0 | 放量+价不涨（量>1.2x+近3日涨幅<3%） | **放量滞涨** ⚠️ | 成交量放大但股价不涨，高位换手密集 | **场景描述**，非独立信号 |
| >0 | >0 | 缩量+横盘不创新高（量<0.8x+波动<5%） | **缩量滞涨** ⚠️ | 成交量萎缩且价格停滞，需求逐步衰竭 | **场景描述**，非独立信号 |
| >0 | >0 | ratio > 1.8 | **加速** | 斜率急剧变陡，拉升阶段（需D方案排除整理后突破） | 正常上行 |
| >0 | >0 | ratio < 0.4 | **缩量整理**（量<0.8x+幅度>3%）或 **上行** | 斜率极低但无异常量价 | 蓄力或慢涨 |
| >0 | >0 | 0.4 ≤ ratio ≤ 1.8 | **上行** | 斜率正常，趋势健康 | 正常上行 |
| >0 | <0 | — | **转弱** | 之前涨、刚拐头向下，危险信号 | 右侧卖出 |

> **重要说明：**
> - **放量滞涨** 和 **缩量滞涨** 是**信号驱动的场景描述词**，不是独立卖出信号
> - 它们的角色是「预警」，告诉交易者"这里需要警惕"
> - 真正的卖出决策需要**融合判定**：放量滞涨+次日确认/需求衰竭高分/向下反转等
> - 详见[融合判定引擎](../design-doc)中的 `_keypoint_direction()` 对这两个阶段的处理

##### 下降趋势阶段判定

| s1 | s2 | ratio | 阶段 | 含义 |
|:--|:---|:------|:----|:-----|
| <0 | <0 | >1.8 | **加速跌** | 斜率急剧变陡，加速下跌 |
| <0 | <0 | ≤1.8 | **下行** | 正常下跌趋势 |
| <0 | >0 | — | **转强** | 之前跌、刚拐头向上 |

##### s1 < 0 && s2 > 0 → 转强（之前跌，刚拐头）
##### s1 > 0 && s2 < 0 → 转弱（之前涨，刚拐头）

### C方案：V反修正

当检测到ratio > 1.8（疑似加速）时，检查近5日是否有急跌阴线（跌幅>3%）导致V反。

```python
for ci in range(1, len(check)):
    if (check[ci] - check[ci-1]) / check[ci-1] < -0.03:    # 急跌>3%
        n_after = len(check) - ci
        start_i = max(1, len(e10_last) - n_after + 1)
        if start_i < len(e10_last) - 2:
            s1_adj = _reg_slope(e10_last[start_i:])  # 用急跌后的EMA10重新算s1
            if s1_adj > 0:
                ratio = s2_val / s1_adj
        break
```

### D方案：整理后突破排除

在ratio仍>1.8的情况下，检查最近是否有过回调且守住了EMA20。有确认过程的突破不算加速。

```python
# 取K线[-23:-3]段，找最近局部低点
for i in range(len(w)-1, len(w)-8, -1):
    if w[i] > w[i-1]:   # 股价变高，前一个是低点
        pull = (peak - trough) / peak * 100
        if pull > 5:    # 回踩>5%
            if trough > ema20[trough_idx] * 0.98:    # 守住EMA20
                ratio = 1.0   # 整理后突破→不算加速，降为上行
        break
```

### 缩量整理区分（区分"缩量整理"、"缩量滞涨"和"放量滞涨"）

当ratio < 0.4时，需要根据**原文（量价原理）** 区分为三种情况，而非之前的两种：

| 条件 | 缩量整理 | 缩量滞涨 | 放量滞涨 |
|:----|:---------|:---------|:---------|
| ratio | <0.4 | <0.4 | <0.4 |
| **量能(末3/前10)** | **缩量<0.8x** | **缩量<0.8x** | **放量>1.3x** |
| **价形态** | 有回踩动作，价在EMA10上 | **横盘/平顶，不再创新高** | **窄幅震荡，实体<1.5%** |
| 全天振幅 | — | — | **<3%** |
| 原文依据 | — | 需求不足→僵持→平顶/圆弧顶 | 量价背离→主力出货 |
| 供需含义 | 正常回调，供应萎缩 | 需求不足，供需僵持 | 需求仍在但推不动价 |
| **操作** | ✅ 持股等突破 | 🟡 **警惕，注意平顶风险** | 🔴 **高度警惕，考虑减仓** |

```python
if volumes and len(volumes) >= 13:
    vol_last3 = sum(volumes[-3:]) / 3
    vol_prev10 = sum(volumes[-13:-3]) / 10
    if vol_prev10 > 0:
        vol_ratio = vol_last3 / vol_prev10
        
        # 放量滞涨（原文5.6节：成交量放大+股价不再上涨+窄幅）
        if vol_ratio > 1.3:
            cur, op = closes[-1], opens_p[-1]
            hi, lo = highs[-1], lows[-1]
            body_pct = abs(cur - op) / op if op else 0
            amp = (hi - lo) / lo if lo else 0
            if body_pct < 0.015 and amp < 0.03:
                return '放量滞涨'
        
        # 缩量 + 价在EMA10上 + 有回踩低点 → 缩量整理
        if vol_ratio < 0.8 and closes[-1] > e10_last[-1]:
            # 检查是否有回踩动作（近N天有低点）
            if len(closes) >= 10:
                recent_low = min(closes[-10:])
                recent_high = max(closes[-10:])
                # 有回踩（低点<近期高位*0.97）→ 缩量整理
                if recent_low < recent_high * 0.97:
                    return '缩量整理'
            # 无回踩 + 横盘 → 缩量滞涨
            return '缩量滞涨'
return '滞涨'  # fallback（原"滞涨"保留作为兼容）
```

## 第6章 关键支撑/阻力位

### 概述

买点（突破判定）和卖点（止损位）都基于关键支撑/阻力位的计算。

**代码位置：** `backend/core/buy_point_detection.py` → `_find_support_levels()` / `calc_stop_loss()`

### 支撑位计算

```python
def _find_support_levels(klines, idx):
    # 1. 扫描所有历史"突破点"：close 突破前10日最高价且收阳
    # 2. 突破点的前10日最高价 = 关键支撑位
    # 3. 取距收盘≥1.5%的最近支撑
    supports = []
    for i in range(10, idx+1):
        if closes[i] > max(highs[i-10:i]) and closes[i] > opens[i]:
            support_price = max(highs[i-10:i])
            if support_price < closes[-1]:
                supports.append(support_price)

    supports = sorted(set(supports), reverse=True)
    for s in supports:
        if (closes[-1] - s) / closes[-1] >= 0.015:
            return s
    return min(highs[-20:])   # 兜底：20日最低
```

### 阻力位（突破判定基准）

```python
# 关键阻力位：最近在上的波峰（前高），而非简单的前10日最高
resistance = _find_resistance_levels(kls, idx)
is_breakout = resistance and close > resistance   # 突破真前高
```

### 止损位计算

```python
def calc_stop_loss(klines, idx):
    # 优先：支撑位 × 0.98（支撑下方2%防假破）
    support = _find_support_levels(klines, idx)
    if support:
        return round(support * 0.98, 2)

    # 兜底：EMA20 × 0.97（EMA下方3%）
    ema20 = ema_list(closes, 20)
    return round(ema20[-1] * 0.97, 2)
```

### 概述

### 缩量阈值（三层感知）

```python
def _shrink_threshold(market_position, is_main_line):
    base = {
        '偏波峰': 0.9, '波中偏上': 0.8,
        '波中': 0.7, '波中偏下': 0.6, '偏波谷': 0.5,
    }.get(market_position, 0.7)
    adj = base * 1.05 if is_main_line else base * 0.95  # 主线系数
    return adj
```

### 突破多维评分

```python
def _breakout_score(close, prev_close, prev_10d_high, vol_ratio, body_ratio, ...):
    score = 0
    # ① 突破幅度（close相对前高的涨幅）
    if (close - prev_10d_high) / prev_10d_high > 0.03: score += 2  # 3%+
    elif (close - prev_10d_high) / prev_10d_high > 0.01: score += 1 # 1%+
    # ② 当日涨幅
    if (close - prev_close) / prev_close > 0.05: score += 2  # 大涨5%+
    elif (close - prev_close) / prev_close > 0.03: score += 1
    # ③ 量比 > 1.2（硬条件）
    if vol_ratio > 2.0: score += 2
    elif vol_ratio > 1.2: score += 1
    else: return (False, 0, ...)  # 量比不足，直接否决
    # ④ 实体占比
    if body_ratio > 0.8: score += 1
    # 总分 ≥ 5 才算有效突破
    return (score >= 5, score, ...)
```

**硬条件：** 量比 > 1.2

### 买点类型和条件

#### 类型1：中继买点（上涨趋势 + 缩量回踩到位）

```
structure == '上涨趋势' AND stage NOT IN ('加速','滞涨','转弱')
AND 缩量(vol_ratio < shrink_th)
AND (地量 或 小实体-3%≤涨幅≤2%)
AND 回踩到位(check_pullback通过)
→ 中继买点, score=3
```

`_check_pullback()` 检查价格是否回踩到EMA5/EMA10/EMA20附近。

#### 类型2：中继买点（区间底部 + 缩量企稳）

```
structure == '区间震荡' AND stage == '区间底部'
AND 缩量(vol_ratio < shrink_th)
→ 中继买点, score=3
```

#### 类型3：突破买点（区间顶部 + 放量突破）

```
(structure == '区间震荡' AND stage == '区间顶部' AND is_breakout)
OR 上涨趋势中的有效突破
AND 突破评分 ≥ 5
AND 近3日无重复突破买点(防重复买入)
→ 突破买点, score=4
```

#### 排除规则

```
structure == '上涨趋势' AND stage in ('加速','滞涨','转弱')
→ 不产生买点（阶段→卖出优先于形态→买入）
```

def check_trailing_take_profit(buy_price, cur_price, peak_price):
    peak_ret = (peak_price - buy_price) / buy_price * 100
    cur_ret = (cur_price - buy_price) / buy_price * 100
    if peak_ret > 5 and (peak_ret - cur_ret) > 10:
