# 手机 APP 方案

> 目的：把现有的网页系统做成手机端可访问，体验接近原生 APP
> 状态：💤 待讨论

---

## 1. PWA — 最轻量方案

**核心思路：** 前端加几行配置，手机浏览器→添加到主屏幕→看起来像 APP

**优点：**
- ✅ 不需要写新代码
- ✅ 不需要应用商店审核
- ✅ 更新即生效（下次打开自动拉最新页面）
- ✅ 支持离线缓存（配置好 service worker 后）

**需要改的：**
1. `manifest.json` — APP名称、图标、主题色、启动屏
2. 512x512 和 192x192 的图标各一张
3. 几行 service worker 注册代码

**缺点：**
- ⚠️ 没有推送通知（技术上可以但和微信消息冲突）
- ⚠️ iOS 上 PWA 支持不如 Android 完善
- ⚠️ 无法上架 App Store

**适合：** 快速验证需求，你自己用，或者给同学用

---

## 2. Capacitor — 半原生方案

**核心思路：** 用现有前端页面包一个 WebView 壳，生成 iOS/Android 安装包

Ionic 团队（Capacitor 的开发者）：

```
你的前端页面（不需要改）
       ↓
  Capacitor 壳（WebView 容器）
       ↓
   iOS .ipa  ← Xcode 生成
   Android .apk ← Android Studio 生成
```

**好处：**
- ✅ 前端代码几乎不用改
- ✅ 可以通过应用商店分发
- ✅ 可以用原生 API（相机、推送通知、指纹等）

**代价：**
- ⚠️ 需要一台 Mac 来打包 iOS 版本（Xcode 只跑在 Mac 上）
- ⚠️ Apple Developer 年费 ¥688
- ⚠️ 安卓可以自己签名分发，iOS 必须通过 App Store 审核

---

## 3. 两个方案的对比

| | PWA | Capacitor |
|:--|:---|:----------|
| 开发成本 | 几十分钟 | 1-2天 |
| 上架商店 | ❌ 不能 | ✅ 可以 |
| 安卓体验 | ✅ 好 | ✅ 好 |
| iOS体验 | ⚠️ 一般 | ✅ 好 |
| 离线缓存 | ✅ 支持 | ✅ 支持 |
| 推送通知 | ⚠️ 有限 | ✅ 原生 |
| Mac 需要 | ❌ 不需要 | ✅ 需要 |
| 年费 | 0 | ¥688（苹果）|

---

## 4. 推荐路线

```
Phase 1: PWA（几个小时搞定）
→ 你现在就能用
→ 同学也能直接访问

Phase 2: 如果觉得体验不够
→ 做 Capacitor 打包
→ 上架安卓商店 + 苹果商店（如果你有 Mac）
```
