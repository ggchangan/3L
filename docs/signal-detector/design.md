# 3L六大关键信号检测系统 设计文档

> 基于简放《量价原理》第5.6节"关键信号"原文量价语言，将自然语言规则可编程化。

---

## 〇、设计思想

3L交易体系中的六大关键信号已由简放在《量价原理》中以自然语言明确定义，每个信号都有完整的量价行为描述。本系统的目标不是"发明新信号"，而是**忠实地将原文规则翻译为可编程的检测算法**，并通过回测验证翻译的准确性。

**核心原则**：
- 每条判定规则必须有原文依据，标注出处
- 参数阈值（如"成交量明显放大"=几倍）通过回测优化，不拍脑袋
- 置信度为连续值（0-100），不做二值判定，让上层系统自行决定阈值
- 先独立检测每个信号，后考虑信号间的叠加/优先级关系

## 一、数据输入格式

```python
Kline = {
    'date': str,       # '20260304'
    'open': float,
    'close': float,
    'high': float,
    'low': float,
    'volume': float,   # 成交量（股）
}
```

输入：60-200天日K线列表，**按日期升序排列**（旧→新）。
每个检测函数取截至当前日的数据，模拟历史时刻的判定。

## 三、九大信号检测算法

本系统实现《量价原理》5.6节全部9个可操作信号 + 平衡信号概念。

### 3.0 信号×结构绑定关系

每个信号**只在其绑定的结构下有效**，这是在代码中以否决项实现的。脱离结构的信号没有意义。

| 信号 | 绑定的结构 | 代码否决项 | 与原文对照 |
|:----|:----------|:---------|:---------|
| **向上突破** | 区间震荡→突破顶部 | `detect_range_trade() == False → 否决` | 5.6(1) |
| **向下突破** | 区间震荡→跌破底部 | `detect_range_trade() == False → 否决` | 5.6(1) |
| **向上反转** | 下跌趋势→反转向上 | `detect_trend() != 'down' → 否决` | 5.6(2) |
| **向下反转** | 上涨趋势→反转向下 | `detect_trend() != 'up' → 否决` | 5.6(2) |
| **需求衰竭** | 上涨趋势末端→需求耗尽 | `detect_trend() != 'up' → 否决` + BIAS20 > 8% | 5.6(3) |
| **供应衰竭** | 下跌趋势末端→供应耗尽 | `detect_trend() != 'down' → 否决` | 5.6(3) |
| **上涨中继** | 上涨趋势中→缩量回踩 | `detect_trend() != 'up' → 否决` | 5.6(4) |
| **下跌中继** | 下跌趋势中→缩量反弹 | `detect_trend() != 'down' → 否决` | 5.6(4) |
| **区间震荡中继** | 区间震荡中→顶底滞涨 | `detect_range_trade() == False → 否决` | 5.6(4) |
| **平衡信号** | 以上皆否 | 无信号触发时的默认状态 | 5.6(4) |

### 3.0.1 信号×结构的可调参数

下表列出已验证有效的次生结构维度（通过回测发现，非原文规定）：

| 信号 | 次生有效维度 | 胜率改善 | 备注 |
|:----|:-----------|:--------|:----|
| **需求衰竭** | 非主线板块可做逃顶 | 52%跌 vs 全量45%跌 | 主线板块上60%继续涨，无效 |
| **需求衰竭** | 90分以上所有板块 | 69%跌 | 高分信号独立有效 |
| **上涨中继** | 主线板块效果更佳 | 65%+ vs 非主线55% | 主线趋势延性好 |
| **供应衰竭** | 回踩EMA5的浅回踩 | 60% vs 其他 | 接近EMA5支撑的信号更可靠 |

### 3.0.2 回测原则

因此，本系统的所有回测遵循：
1. **每个信号只在其绑定的结构下回测**（代码中已通过否决项实现）
2. **分层回测**作为补充，寻找次生有效维度
3. **结论标注结构范围**，避免误导

### 信号1：向上突破

**原文出处**：《量价原理》5.6(1)

**判定条件**（逐条编码）：

```
输入：klines[0..n-1]，其中 klines[n-1] 为当前日
输出：{ triggered: bool, confidence: 0-100, detail: str }
```

| # | 原文规则 | 算法实现 | 权重 |
|:-:|:--------|:---------|:----:|
| 1 | 发生在区间震荡中 | `detect_range_trade(klines[:-1])` 判断突破前处于区间震荡 | 否决项 |
| 2 | 股价突破前高（区间顶部） | `close > range_high * 1.01`（突破1%以上） | 30分 |
| 3 | 成交量明显放大 | `volume_ratio > THRESHOLD`（可调，默认1.8倍）| 25分 |
| 4 | 突破前成交量逐步萎缩、波动收窄 | 前10日volume趋势为下降（线性回归斜率负），且前10日振幅均值 < 整体振幅的60% | 20分 |
| 5 | 放量长阳，收盘在相对高位 | 实体占比 > 60%（大阳线），且 close 位于上1/3区间 | 15分 |
| 6 | 突破后验证（回测用） | 2-3日后仍不折回区间内 | 10分（历史回测加分，不参与实时判定）|

**置信度算法**：否决项不通过直接返回0；通过后按权重累加分数。

**可调参数**：
- `volume_threshold`: 1.5 - 3.0（突破日成交量倍率）
- `break_pct`: 0.5% - 2%（突破幅度）
- `pre_shrink_window`: 5-15（突破前缩量观察窗口）

## 五、回测框架

### 5.1 回测方式

对每只股票，在60天数据上**滚动窗口检测**：

```
for i in range(DETECT_START, len(klines)):
    # 仅使用 klines[0:i+1] 的数据模拟历史
    signal = detect_signal(klines[0:i+1])
    if signal.triggered:
        # 记录信号触发时的价格
        entry_price = klines[i].close
        # 看N日后的表现
        for horizon in [3, 5, 10, 20]:
            if i + horizon < len(klines):
                ret = (klines[i+horizon].close - entry_price) / entry_price
                results.append({ 'signal': name, 'entry_date': klines[i].date, ... })
```

### 5.2 统计指标

每个信号单独统计：
- **触发次数**
- **胜率**（买入信号：N日后涨幅>0；卖出信号：N日后跌幅>0）
- **平均盈亏**
- **盈亏比**（平均盈利 / 平均亏损）
- **最大连续亏损次数**
- **不同置信度区间的胜率**（70-80分、80-90分、90+分）

### 5.3 参数调优

通过遍历参数组合找到最优解：
- volume_threshold: [1.5, 1.8, 2.0, 2.5, 3.0]
- break_pct: [0.5%, 1%, 1.5%, 2%]
- 权重系数微调

## 八、结构分层：衰竭信号在3L体系中的应用

### 8.1 为什么需要结构分层

需求衰竭和供应衰竭不是独立交易信号，它们在3L体系中的角色是**波峰/波谷的辅助确认**：

- **需求衰竭** → 出现在上升趋势末端 → 提示可能进入波峰区域
- **供应衰竭** → 出现在下降趋势末端 → 提示可能进入波谷区域

如果脱离结构（大盘位置、板块主线、个股阶段）单独使用这两个信号，有效性会被大幅稀释——因为：
- 上涨中继的短暂加速也会触发需求衰竭（但趋势健康）
- 下跌中继的恐慌也会触发供应衰竭（但不是底部）

### 8.2 需求衰竭的分层预期

| 大盘结构 | 板块状态 | 个股位置 | 预期有效性 |
|:--------|:--------|:--------|:---------:|
| 波峰区域 | 主线/热点 | 加速末端 | ⭐⭐⭐⭐⭐ 逃顶可靠 |
| 波中区域 | 主线 | 上涨中继 | ⭐⭐ 信号偏弱，趋势延续 |
| 波中区域 | 非主线 | 正常上涨 | ⭐ 大概率假信号 |
| 波谷区域 | 任何 | 任何 | ⭐ 跌势中需求衰竭无意义 |

### 8.3 供应衰竭的分层预期

| 大盘结构 | 板块状态 | 个股位置 | 预期有效性 |
|:--------|:--------|:--------|:---------:|
| 波谷区域 | 主线回调 | 波谷企稳 | ⭐⭐⭐⭐⭐ 抄底可靠 |
| 波谷区域 | 非主线 | 恐慌末端 | ⭐⭐⭐ 可能反弹 |
| 波中区域 | 任何 | 下跌中继 | ⭐ 大概率继续跌 |
| 波峰区域 | 任何 | 任何 | ⭐ 上涨趋势中供应衰竭无意义 |

### 8.4 实现策略

在回测框架中增加结构分层能力：
1. **板块主线判定**：从 `mainline_history.json` 获取每日主线板块列表
2. **个股行业映射**：从 `stock_industry_map.json` 获取个股所属行业
3. **大盘位置**：从中证全指的波峰波谷判定获取
4. **分层回测**：按以上维度分组统计每个信号的胜率变化

### 8.5 预期结果

经过结构分层后，信号在匹配的结构下胜率应显著高于全量水平：
- 需求衰竭(加速)在"主线上涨+波峰区域"下：5日跌幅预期 > 60%
- 供应衰竭在"主线回调+波谷区域"下：5日涨幅预期 > 65%

## 九、关键点 × 关键信号：信号应用框架

> 本节内容来源：《量价原理》5.7 "只在关键点和关键信号交易"

### 9.1 辩证关系

信号检测器产出的是"关键信号"，但**只有关键信号是不够的**。原书5.7节明确了两者的辩证关系：

| 维度 | 属性 | 含义 | 系统对应 |
|:---|:---:|:-----|:--------|
| **关键点** | 静态·位置 | 波峰、波谷、前高、前低、均线支撑/压力 | 结构判定 + 波峰波谷识别 |
| **关键信号** | 动态·行为 | 突破、反转、衰竭、中继的量价行为 | 九大信号检测器 |

**两者的关系**：
- **只有关键点没有信号** → 位置到了但行为未确认 → 等待（平衡信号 = 不操作）
- **只有信号没有关键点** → 行为出现在非关键位置 → 大概率假信号（如需求衰竭在波中区域）
- **关键点 + 关键信号同时出现** → 概率优势最大 → 执行操作

这就是为什么我们之前在8.1节得出结论：需求衰竭在非主线板块上有效、在主线上无效——因为主线上的关键点（波峰）还没到，需求衰竭只是中继加速的假信号。

### 9.2 五步交易程序

原书第4997-5005行给出了完整的交易流程，也是本系统信号层 → 操作层的映射标准：

```
步骤1：识别当前结构走势
       ↓ 是上升趋势、下降趋势还是区间震荡？
步骤2：判断走势是延续还是转折
       ↓ 当前位置在波峰、波谷还是波中？
步骤3：识别关键点
       ↓ 波峰/波谷/前高/前低/均线支撑
步骤4：根据关键点识别关键信号
       ↓ 突破？反转？衰竭？中继？
步骤5：操作
       ↓ 买入、卖出、持有、等待
```

### 9.3 系统映射

本系统的信号检测器覆盖了步骤4的所有关键信号类型：

| 信号大类 | 信号名称 | 性质 |
|:--------|:--------|:----|
| **突破信号** | 向上突破、向下突破 | 改变供需格局 |
| **反转信号** | 向上反转、向下反转 | 改变供需格局 |
| **衰竭信号** | 需求衰竭、供应衰竭 | 改变供需格局 |
| **中继信号** | 上涨中继、下跌中继、区间震荡中继 | 维持原走势 |
| **平衡信号** | 以上皆否时触发 | 不操作，等待 |

**平衡信号**不是单独实现的检测器，而是当其他9个信号均未触发时的**默认状态**。原书4833-4845行明确："平衡信号不是操作信号，而是'不操作'信号，是等待信号。"

### 9.4 信号优先级

当多个信号同时触发时，按以下优先级处理：
1. **突破信号** > **反转信号** > **衰竭信号** > **中继信号**
2. 同级别取置信度高者
3. 平衡信号权重最低，仅在无其他信号时生效

这是因为突破和反转改变供需格局，对操作的指导意义大于维持原走势的中继信号。
