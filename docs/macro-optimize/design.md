## 1. 背景与问题

**现状：** 宏观环境页有"外围美股映射"区块，展示与A股有供应链关系的美股。当前问题：
- 映射表23只美股，只拉了16只实时行情
- 涨跌幅就是数字，没有自动标记异常
- 美股大跌时不知道原因，需要手动去查

**痛点：** 美股异动直接影响A股对应板块的开盘预期（如英伟达大跌→A股算力板块承压），但页面没有预警机制，异动原因需要外部搜索。

**目标：** 补全所有映射美股实时数据，自动检测异常涨跌并提供原因分析入口。

---

## 2. 设计思想

### 2.1 核心理念

外围映射的核心价值是"美股异动 → 影响A股开仓判断"。异动不标记等于没数据，只标记不分析等于只做了一半。

### 2.2 方案选择

| 候选方案 | 优点 | 缺点 | 结论 |
|:--------|:----|:----|:----:|
| 只补行情不分析 | 最小改动 | 异动无上下文，用户还要自己去查 | ❌ |
| 补行情+异动标记+AI分析入口 | 完整闭环，按需调用 | 多一个API | ✅ |
| 自动推送异动通知 | 主动预警 | 要加cron/wxpusher，复杂度高 | ❌（后续可加） |

### 2.3 设计原则

- **页面内闭环：** 不跳转外部网站，异动原因直接在页面内查看
- **按需分析：** 不提前拉新闻消耗资源，用户点击「分析原因」才触发
- **异动分级：** ±3%/±5%两个阈值，避免过度提示

### 2.4 屏幕内外

- **做什么：** 补全23只美股行情、自动检测异动、提供AI原因分析
- **不做什么：** 不做推送通知、不做历史异动统计、不自动轮询新闻

### 3.3 异动判定规则

```python
def get_alert_level(change_pct):
    abs_chg = abs(change_pct)
    if abs_chg >= 5:
        return 'warning'   # 🔴
    elif abs_chg >= 3:
        return 'caution'   # ⚠️
    return None
```

## 4. 系统设计

### 4.1 后端改动

**macro_service.py** — 修改范围：
1. 补齐所有23只美股代码到腾讯API拉取列表
2. 遍历映射表，逐只匹配实时行情+计算异动等级
3. 返回 `abnormal_alerts` 数组

**新增：macro_analysis_service.py** — 美股异动新闻搜索：
1. 接收股票代码
2. 搜索相关新闻（web搜索 或 预设新闻源）
3. 返回新闻时间线

**新增API路由：**
```
GET /api/macro/analyze-abnormal?code=NVDA
```

### 4.2 前端改动

**Macro.tsx：**
1. 美股卡片根据 `level` 字段加背景色/边框
2. 新增「⚠️ 外围异动监测」区块（条件渲染，无异常时隐藏）
3. 每条异动加「🔍 分析原因」按钮
4. 分析结果弹窗组件

**Macro.css：**
- `.alert-warning` — 黄色边框/标签
- `.alert-danger` — 红色边框/标签
- `.analysis-modal` — 分析结果弹窗样式

### 4.3 交互流程

```
用户点击 [分析原因]
  → 前端 POST /api/macro/analyze-abnormal?code=NVDA
  → 后端搜索新闻，返回时间线+摘要
  → 前端弹窗展示分析结果
  → 用户可继续追问（回到聊天窗口）
```
