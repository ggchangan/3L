## 1. 背景与问题

### 现状

复盘页面当前由5个独立区块组成：

| 区块 | 内容 | 问题 |
|:----|:-----|:-----|
| STEP 1 大盘周期 | 大盘定性、仓位水位 | ✅ 正常 |
| STEP 2 主线排名 | 行业前10按20日涨幅排序 | ❌ 只看涨了多少，不知道处于什么阶段 |
| STEP 3 持仓个股 | 量价择时诊断 | ❌ 没有标注所属概念/方向 |
| STEP 4 自选股信号 | 买点信号 | ❌ 没有标注所属概念/方向 |
| PLAN 交易计划 | 持仓操作+买入优先级 | ❌ 没有按方向机会排序 |

### 痛点

**① 主线排名的信息量不够**

用户看到"半导体排第3，20日+17.6%"，能知道它涨了很多。但不知道：
- 它是一个波谷回调的优质买点？（主线+波谷 = 最好机会）
- 还是已经涨到波峰了该减仓？（主线+波峰 = 见顶风险）
- 还是趋势健康的在涨？（主线+上涨 = 持有不动）

同样，不在前10的板块用户就忽略了，但其中可能有**波谷确认、未来可能变成主线的方向**。

**② 个股和方向之间没有关联**

持仓股/自选股只知道自己的买点信号和止损，但不知道：
- 这个股属于哪个概念/行业？
- 该概念/行业现在处于什么阶段（波谷/波峰/上涨/下跌）？
- 这个方向整体是机会还是风险？

**③ 交易计划没有方向优先级**

PLAN 列出了所有持仓操作和买入优先级，但买入优先级只按个股信号排序，没有考虑"这个方向整体好不好"——应该优先做主线回调方向的机会。

### 目标

在复盘页面上建立一条完整的逻辑链：

```
大盘周期 → 方向机会分类(行业/概念×阶段) → 个股买卖点(关联方向)
```

用一句话说：**用户打开复盘页面，一眼就能知道当前市场哪些方向值得做、哪些该回避、自己的持仓/自选分别落在哪个方向里。**

## 2. 设计思想

### 2.1 核心理念

**机会的分类比排序更重要。**

当前按20日涨幅排名，本质是"谁涨得多谁排前面"。但交易不是这么做的——一个涨了27%但已到波峰的板块，不如一个回调到波谷、量缩到位的主线板块有操作价值。

3L体系的核心理念是**"先识别趋势，再用乖离率判断位置"**。应用到方向分类上就是：

| 主线状态 | 阶段 | 含义 | 操作价值 |
|:--------|:----|:-----|:--------|
| ✅ 是主线 | 波谷 | 主线回调到位，趋势未变 | ⭐⭐⭐ 最好买点 |
| ❌ 非主线 | 波谷(vl≥3) | 可能是未来主线，提前布局 | ⭐⭐ 潜在机会 |
| ✅ 是主线 | 上涨/波中 | 趋势健康，持有为主 | ⭐ 持有 |
| ✅ 是主线 | 波峰 | 涨过头了，注意见顶 | ❌ 减仓/回避 |
| ❌ 非主线 | 其他 | 无明确信号 | — 观察 |

**优先级顺序：** 主线回调 > 潜在主线 > 趋势延续 > 见顶风险 > 观察

### 2.2 方案选择

候选方案对比：

| 方案 | 优点 | 缺点 | 结论 |
|:----|:----|:-----|:----:|
| A. 按机会类型分组（本方案） | 用户一眼看到该做什么 | 需要合并主线+概念两套数据 | ✅ |
| B. 在现有排名上加阶段标签 | 改动小 | 仍然按涨幅排，不直观 | ❌ |
| C. 新增独立"机会发现"页面 | 不干扰复盘页 | 用户要多切一个页面，割裂 | ❌ |

**为什么选A：** 复盘页的核心目的是"看完就知道明天怎么操作"。按机会类型分组后，用户天然地按照优先顺序看——最先看"主线回调"，最后看"见顶风险"。信息流和决策流一致。

### 2.3 设计原则

1. **机会类型一眼可辨** — 用emoji+颜色，不依赖用户读文字判断
2. **所有方向覆盖** — 375个概念和90个行业都经过分类，不只是前10名
3. **个股关联方向** — 每一只持仓/自选股都知道自己属于哪个机会类型
4. **Plan按机会排序** — 买入优先级 = (机会类型权重 × 个股信号强度)

### 2.4 屏幕内外

**做什么：**
- 复盘页 STEP 2 重构：按"主线回调/潜在主线/趋势延续/见顶风险"分组展示行业和概念
- 保留 [行业] [概念] Tab 切换
- 每个分组内按20日涨幅排序
- 每行显示：板块名、今日涨跌、20日涨幅、5阶段标签、机会类型标签
- 个股卡片加"关联方向"行，显示所属概念/行业及该方向的机会类型
- PLAN 买入优先级按机会类型排序

**不做什么：**
- 不改动大盘周期判定逻辑（STEP 1）
- 不改动个股买点信号计算逻辑（STEP 3/4）
- 不新增数据库/API端点，复用现有数据源
- 本期不做盘中实时轮动监测（后续单独做）

## 3. 数据模型

### 3.1 核心数据结构

每个行业/概念的完整信息：

```python
{
    'name': '半导体',           # 板块名
    'chg_20d': 17.55,           # 20日涨幅(%)
    'chg_1d': -6.22,            # 今日涨跌幅(%)
    'stage': '波谷',            # 5阶段: 波谷/波峰/上涨/下跌/波中
    'vl_score': 4,              # 波谷评分 0~5
    'volume_ratio': 0.65,       # 量比
    'is_mainline': True,        # 是否主线(前5)
    'is_secondary': False,      # 是否次线(6-10)
    'opportunity': '主线回调',   # 机会类型
    'persistence_days': 5,      # 持续天数
}
```

机会类型枚举：

| 枚举值 | 条件 | emoji | 颜色 |
|:------|:----|:-----|:----|
| `主线回调` | is_mainline=True AND stage='波谷' | 🎯 | 红色 |
| `潜在主线` | is_mainline=False AND stage='波谷' AND vl_score≥3 | 🔮 | 青色 |
| `趋势延续` | is_mainline=True AND stage in ('上涨','波中') | 📈 | 绿色 |
| `见顶风险` | is_mainline=True AND stage='波峰' | ⚠️ | 橙色 |
| `回调中` | is_mainline=True AND stage='下跌' | 📉 | 灰色 |
| `次线机会` | is_secondary=True AND stage='波谷' | 🔮 | 青色 |
| `--` | 其他 | — | 灰色 |

### 3.2 数据流

```
sector_daily.json
    │
    ├── industries → get_mainline_data()
    │       │         遍历每个行业k线
    │       │         → 计算 chg_20d, chg_1d
    │       │         → judge_concept_wave(klines) 得出 stage
    │       │         → 结合排名(is_mainline) 得出 opportunity
    │       ▼
    └── concepts  → get_concept_mainline_data()
            │         同上逻辑
            ▼
    review API → 返回 mainline 含 lines/concept_mainline
                        每个 item 含 stage/opportunity
            │
            ▼
    前端 MainlineSection → 按 opportunity 分组渲染
    前端 HoldingsReview   → 关联方向标注
    PLAN                 → 按机会类型排序
```

### 4.2 核心算法

**机会分类函数：**

```python
def classify_opportunity(is_mainline: bool, is_secondary: bool,
                         stage: str, vl_score: float) -> str:
    """根据主线状态和5阶段判定机会类型"""
    if is_mainline or is_secondary:
        if stage == '波谷':
            return '主线回调' if is_mainline else '次线机会'
        elif stage == '波峰':
            return '见顶风险'
        elif stage in ('上涨', '波中'):
            return '趋势延续'
        elif stage == '下跌':
            return '回调中'
        return '主线观察'
    else:
        # 非榜板块：只有波谷+高评分才算潜在主线
        if stage == '波谷' and vl_score >= 3:
            return '潜在主线'
        return '--'
```

**阶段判定复用：**

直接复用 `concept_wave_service.judge_concept_wave(klines)`，输入任意板块（行业/概念）的K线数组，返回 stage/vl_score/volume_ratio 等所有字段。该函数不依赖外部状态，纯计算。

## 6. 附录

### 6.1 替代方案

**方案B（在现有排名上加阶段标签）** 被否决的原因：如果只是加一列"阶段"，用户仍然看到的是按涨幅排序的列表，不会优先注意到"主线回调"的半导体——它排在"趋势延续"的元件后面。信息呈现方式决定了用户的注意力分配。

### 6.2 开放问题

1. 行业板块目前没有波谷评分（`judge_concept_wave` 虽然能复用，但行业和概念的量价特征不同，是否需要微调参数？）→ 先复用，后续评估
2. 概念板块共375个，分组后每个组可能很多（如"趋势延续"组可能20+个），是否要限制每个组最多显示10个？→ 显示全部，但"其他观察"组默认折叠
