# 结构判断 + 阶段判断（3L盯盘版）

## 结构判断

### 算法

取最近60个交易日的收盘价计算完整EMA10，取末15根找极值位置。

```python
e10_full = ema_list(closes, 10)    # 全量60天计算EMA10！
e10 = e10_full[-15:]                # 取末15根
n = len(e10)  # 15
max_pos = max(range(n), key=lambda i: e10[i])
min_pos = min(range(n), key=lambda i: e10[i])
first_q = n // 4   # 3
last_q = n - 1 - n // 4  # 11

if max_pos >= last_q and min_pos <= first_q:
    structure = '📈 上涨趋势'
elif min_pos >= last_q and max_pos <= first_q:
    structure = '📉 下降趋势'
else:
    structure = '➡ 区间震荡'
```

### ⚠️ 常见坑：EMA10不能从15根K线独立计算

**错误做法：** `e10 = ema_list(closes[-15:], 10)` ← 旧版代码踩过的坑

**根因：** EMA是指数移动平均，起始值必须继承历史数据。如果只从15根K线算，第1根EMA10等于该根收盘价（因为没有前值），导致：
- 起始15天的最高价出现在第1根 → 最大值永远在头部 → 统计偏差
- 起始15天的最低价出现在最后 → 最小值永远在尾部
- 结果：**系统性高估下降趋势**（如深圳华强从底部反弹时仍误判为下降趋势）

**修复（2026-05-21）：** 全量计算EMA10后再取末15根 —— `ema_list(closes, 10)[-15:]`

`scripts/ema_utils.py` 已修复。

### 算例对比

| 场景 | 错误算法（15根独立算） | 正确算法（全量计算取末15根） |
|:----|:--------------------|:-------------------------|
| 急跌后反弹（深圳华强） | EMA10=37.77→34.60, 最大值在[0] → 下降趋势 | EMA10=31.98→34.25, 最大值在[9] → 区间震荡 |
| 整体上涨中回调（拓普集团） | EMA10方向比0.39 → 区间震荡 | EMA10=64.47→? 极值在头部/尾部 → 上涨趋势 |

### 判定规则

| EMA10最高位置 | EMA10最低位置 | 结论 |
|:-------------|:-------------|:-----|
| 后25%（尾部） | 前25%（头部） | 📈 上涨趋势 |
| 前25%（头部） | 后25%（尾部） | 📉 下降趋势 |
| 其他位置 | — | ➡ 区间震荡 |

### ⚠️ EMA10固有滞后性

EMA10是指数移动平均，天然滞后于实际价格。具体表现：

- **价格已见顶回落**，EMA10可能还在继续走高（滞后约2-3天）
- **价格已触底反弹**，EMA10可能还在继续走低

这是EMA10作为均线的正常特征，不是bug。滞后会导致：
- 先涨后跌的板块：可能在价格见顶后2-3天仍显示上涨趋势，然后才转为区间震荡
- 先跌后涨的板块：同理

**光伏设备案例：** 价格在5天前到顶（10040），EMA10直到今天还在涨（因为EMA10平滑了之前的涨幅）。这是EMA10的正常滞后行为。

## 阶段判断

用EMA10线性回归斜率判断，s1=末15根整体斜率（趋势基准），s2=末3根斜率（近期动量，更敏感）。

```python
e10_full = ema_list(closes, 10)        # 全量60天计算EMA10
e10_last = [v for v in e10_full[-15:] if v is not None]  # 取末15根
s1 = _reg_slope(e10_last)              # 末15根整体斜率（趋势基准）
s2 = _reg_slope(e10_last[-3:])         # 末3根斜率（近期动量，更敏感）
```

**⚠️ 坑：阶段判断必须用与结构判断相同的窗口（末15根），不能用全部EMA10。**

### 区间震荡 — 用关键点支撑替代15日极值算位置

**背景：** 区间震荡的阶段判定原用最近15日最高/最低价算分位。但15日内有**单根异常大阳线或长上影**时，会拉升区间顶，导致实际在区间中部的价格被误判为顶部，进而触发 `stock-action-judgment` 的"回避模板②"误判为卖出。

**实例（大族数控 301200）：**
```
15日最低176.37(5/7)  15日最高240.00(5/13单根拉升)
当前224.00
旧算法(15日极值):  (224-176)/(240-176)=74.9% → 区间顶部 ❌ 误判卖出
```

**修复方案：** `support_level`/`resistance_level` 由调用方从K线"突"关键点计算后传入，替代原始15日极值。

```python
# get_stage() 新增参数：support_level, resistance_level
# 优先使用关键点支撑/压力，回退到15日极值
lo = support_level if support_level else min(lows[-15:])
hi = resistance_level if resistance_level else max(highs[-15:])
pct = (cur - lo) / (hi - lo) * 100
if pct < 30:   → 区间底部
elif pct > 70: → 区间顶部
else:          → 区间中段
```

**大族数控修复效果：**
```
新算法(关键点支撑): 区间[220, 240] → (224-220)/(240-220)=20.0% → 区间底部 ✅
```

**支撑选择规则（2026-05-21 修正）：**
1. 找出所有当前价下方的"突"关键点，`support_price` = 突破前10日最高价
2. 从高到低排序（最靠近当前价的优先）
3. 取第一个距当前价 **≥1.5%** 的作为有效支撑
4. 所有突关键点都距当前<1.5%时，**逐档下试**，不直接跳20日最低
5. 全无有效突关键点，才回退到20日最低

**错误历史（2026-05-21）：** `generate_review_data.py` 初期实现中，最近支撑(34.34, 距当前0.2%)被过滤后直接跳到20日最低(29.58)，导致深圳华强从底部(18.6%)被误判为中段(45.8%)。正确行为是试下一档支撑(33.10)。`batch_gen_charts.py` 逻辑本就正确（已有 `deeper` 数组试下一档），但 `generate_review_data.py` 是新写代码踩了同样的坑。

### 上涨趋势 — 加速/滞涨/上行/转弱

```
s1>0, s2>0 (整体趋势向上，近期也在向上)
  ├─ ratio > 1.8 → 🚀 加速（近期斜率远超整体，快速拉升）
  ├─ ratio < 0.4 → 需结合量能区分：
  │   ├─ 量缩80%+价在EMA10上 → 🔄 缩量整理（中继蓄力，可持股/加仓）
  │   └─ 其他                 → ⚠️ 滞涨（警惕出货，减仓/离场）
  └─ 其他        → ↑ 上行（正常健康趋势）
s1>0, s2<0       → 📉 转弱（整体向上但近期已拐头）

**⚠️ 缩量滞涨 vs 中继买点区分（参考 `2026-05-21-shrink-stagnation-vs-relay.md`）：**
同样的缩量（如量比74%），在ratio≥0.4时是「中继买点」，在ratio<0.4时是「缩量滞涨」。
核心区别不是缩量本身，而是EMA10斜率——斜率正常时缩量=供应被吸收，斜率走平时缩量=无人买。
```

**⚠️ V反误判加速（C方案 2026-05-21）：** 当ratio > 1.8时，检查近6根收盘价是否有单日跌幅>3%的急跌。如有，将s1起点从15根改为急跌后重新算，避免急跌后的V型反弹被误判为加速：

```python
if ratio > 1.8 and len(closes) >= 6:
    check = closes[-6:]
    for ci in range(1, len(check)):
        if (check[ci] - check[ci-1]) / check[ci-1] < -0.03:
            n_after = len(check) - ci
            start = max(1, len(e10_last) - n_after + 1)
            if start < len(e10_last) - 2:
                s1_adj = _reg_slope(e10_last[start:])
                if s1_adj > 0:
                    ratio = s2 / s1_adj
            break
```

**实例（伟创电气 688698，2026-05-21 修正）：**
```
5/14 收盘68.88（-3.9%急跌）
原始: s1=0.522(含回调)  s2=1.249(仅反弹)  ratio=2.39→误判加速 ❌
调整: s1_adj=1.150(从急跌后算)  ratio=1.09→上行 ✅
```

### 下降趋势 — 加速跌/下行/转强

```
s1<0, s2<0 (整体趋势向下，近期也在向下)
  ├─ ratio > 1.8 → 📉 加速跌（近期跌速远超整体）
  └─ 其他        → ↓ 下行（正常下跌）
s1<0, s2>0       → 📈 转强（整体向下但近期已拐头向上）
```

### D方案：两步法判断整理后突破（2026-05-22）

判加速(ratio>1.8)前，检查当前加速是否为整理后突破：

**第1步**：加速前20个交易日（排除最近5天拉升段），从最近的高点往后，是否有超过5%的回调，且回调低点守在EMA20上方？
**第2步**：突破时最近3日均量 / 前12日均量 < 1.7？

两项通过 → 整理后突破，降为上行。否则留加速。

```python
w = closes[-25:-5]  # 排除最近5天拉升段
peak_val = max(w)
after_w = w[w.index(peak_val)+1:]
if after_w:
    trough_val = min(after_w)
    pull = (peak_val - trough_val) / peak_val * 100
    if pull > 5:
        # 第1步：守EMA20
        e20 = ema_list(closes, 20)
        trough_abs_i = len(closes) - 25 + peak_i + 1 + after_w.index(trough_val)
        if trough_val > e20[trough_abs_i] * 0.98:
            # 第2步：量比<1.7
            v3 = sum(volumes[-3:]) / 3
            v12 = sum(volumes[-15:-3]) / 12
            if v3 / v12 < 1.7:
                ratio = 1.0  # 判为上行
```

**案例**：
- 多氟多(002407)：W底突破，量比1.55 → 上行✅
- 德业股份(605117)：正常回调后突破，量比1.36 → 上行✅
- 中芯国际(688981)：守位但量比2.06>1.7 → 留加速✅
- 伟创电气(688698)：守位但量比1.75>1.7 → 边界误伤，实际走势健康

**⚠️ 量比阈值1.7的局限（2026-05-22 讨论中）：** 伟创电气(量比1.75)突破了但实际走势健康（底部横盘20天→涨停突破→沿EMA5正常上涨）。1.7阈值对此类"底部横盘后突破"的放量过于严格，因为前12天包含横盘地量导致分母偏小、量比虚高。

**替代方向探讨：乖离率变化速度**：不设固定量比阈值，改为检查价格偏离EMA5的日变化幅度。沿EMA5走的股票乖离率日变化±1~2%，而真正加速的股票一天乖离率跳5%+。这更接近供需本质——"需求是持续进来的还是一天砸完的"。详见 `references/2026-05-22-deviation-change-vs-volume-threshold.md`。

### ⚠️ 半段切分的边界敏感性

s1用15根整体斜率、s2用末3根近期斜率，近期动量更灵敏。相比原来的对半切分，避免了拐点恰好在切分边界上导致的误判。

列线图示意：
```
15根EMA10:  ──────────────┬────
                         ↑ s1=整体趋势方向
3根近期:         ────────┘
                         ↑ s2=近期动量（更敏感）
```

### ⚠️ get_stage() 结果与 buy_point_detection 可能冲突

`get_stage()` 返回的阶段与 `buy_point_detection` 扫出的买点可能互相矛盾：

- 阶段="加速"（→卖出信号）但买点扫描="突破买点"（→买入信号）
- 这是因为阶段基于**EMA10趋势窗口（15根）**，买点基于**单日K线形态**，时间尺度不同

**处理原则（在调用方 `judge_signal()` 中实现）：** 卖出 > 买入 > 持有。阶段判定优先。详见 `stock-action-judgment` skill。
