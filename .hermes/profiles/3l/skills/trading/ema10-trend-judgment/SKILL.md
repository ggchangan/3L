# 结构判断 + 阶段判断（3L盯盘版）

## 结构判断

### 算法（终版：EMA10极值 + 双侧末端校验）

**核心逻辑：** 先以EMA10极值位置做初步判定（平滑噪音），再用双侧末端校验修正滞后问题。

**四层判断流程：**

```python
# 第1层：EMA10极值位置（基础判定）
e10 = ema_list(closes, 10)[-15:]       # 全量60天计算EMA10，取末15根
n = len(e10)
max_pos = max(range(n), key=lambda i: e10[i])
min_pos = min(range(n), key=lambda i: e10[i])
first_q = n // 4  # 3
last_q = n - 1 - n // 4  # 11

if max_pos >= last_q and min_pos <= first_q:
    base = '上涨趋势'
elif min_pos >= last_q and max_pos <= first_q:
    base = '下降趋势'
else:
    base = '区间震荡'

l3 = 最近3根EMA10值（去None）

# 第2层：上涨降级（滞后修正）
# 润泽科技案例：EMA10尚在尾部但价格已跌 → 降级
if base == '上涨趋势':
    if l3连续下降 AND close < 最新EMA10:
        return '区间震荡'

# 第3层：下降升级（末端反弹修正）
# 皓元医药案例：EMA10尚在底部但价格已涨 → 升级
if base == '下降趋势':
    if l3连续上升 AND close > 最新EMA10:
        return '区间震荡'

# 第4层：区间震荡→下降（对称校验）
# 天华新能/宁德时代案例：先涨后跌，EMA10最高在中部，最低在头部
# 收盘新低+EMA10下降+20日跌>3%+非巨涨
if base == '区间震荡':
    c15 = closes[-15:]
    if (收盘最低在后3根 AND 收盘最高在前5根
        AND l3连续下降 AND close < 最新EMA10
        AND 20日涨幅 < -3%    # 排除轻微回调
        AND 60日涨幅 < 50%): # 排除暴涨后回调
        return '下降趋势'
```

## 阶段判定（2026-05-24 R3终版：乖离率变化替代斜率比）

> **中继买点判定已不再依赖 `get_stage()` 的返回值。** 2026-05-24代码改造后，`detect_buy_point()` 直接使用乖离率变化+回踩到位二选一来判断，绕过了stage的"滞涨/转弱"漏判问题。

### 新方案：乖离率变化三段论

```
乖离率(close-EMA10)/EMA10 末3日趋势：
├── >0（扩大）→ 上行/加速
├── <0（收窄）→ 回踩阶段  ← 买点区域
│   ├── 缩量+乖离率±2% → 均线支撑 ✅
│   └── 地量(15%分位) → 供应枯竭 ✅ 不限位置
└── ≈0（横盘）→ 整理阶段
    ├── 缩量 → 中继 ✅
    └── 放量 → 滞涨 ❌
```

### 中继买点完整规则（代码已实现）

```
上涨趋势 + 缩量(量比<动态阈值) + 回踩到位（二选一）:
  ① 乖离率(EMA5/10/20最近)在±2%内 → 哪条近算哪条
  ② 地量(当日量<近20日量能15%分位) → 位置不限
```

### 阴包阳三维判定

```
阴包阳出现 → 逐级判定:
第1层 跌幅: 日涨幅<-5%直接走 />-3%不走 / -5%~-3%看第2层
第2层 支撑: 破关键支撑走 / 没破观察
第3层 量能: 放量(>1.0)加强判断
```

详见 `references/2026-05-24-乖离率变化替代斜率比阶段判定.md`。

### ⚠️ 常见坑

1. **EMA10不能从15根K线独立计算** — 必须全量60天计算后取末15根。见下方「算例对比」。
2. **纯收盘价方案太敏感** — 2026-05-20第4轮曾定稿用收盘价但未实现，2026-05-22尝试后被用户否决（华特气体案例：大跌日导致下降趋势假阳性）。
3. **对称校验的20日< -3%阈值** — 2026-05-22讨论中用户指定。太松会把正常回调误判为下降（如药明康德末3天在反弹），太紧会漏掉慢跌。
4. **60日<50%过滤** — 排除暴涨后的正常回调（如应流股份+233%、生益电子+820%）。这些股票EMA10本身可能判下降趋势，但实际是上涨中的回调。
5. **✅ 中继买点不再依赖stage判定（2026-05-24代码已实现）** — 改用乖离率变化+实体条件+回踩到位三选一。详见 `references/2026-05-24-乖离率变化替代斜率比阶段判定.md`

### ⚠️ 中间窗口局部低点导致上涨趋势误判为区间震荡（2026-05-24）

**案例：德明利(001309) 2026-03-16**
3/13收盘320突破前高，结构=上涨趋势 ✅。3/16收盘352（+10%），结构=区间震荡 ❌。

**根因：** EMA10末15根窗口中有个局部最低点（246.15）在**中间位置(position 5)**，不在前四分之一（fq=3）。算法要求 `min_pos ≤ fq` 才判上涨趋势，min_pos=5 > 3 → 判为区间震荡。

```
3/16 EMA10末15根：
[0] 247.56  [1] 248.14  [2] 249.65  [3] 250.80  [4] 250.03
[5] 246.15  ← 全局最低！在窗口第2四分之一
[14] 295.62 ← 最高

上涨趋势需: max≥11 ✔ AND min≤3 ✗ (min=5) → 区间震荡
```

波浪窗口滑动，早期的一个回调低点滑到了窗口中间，但整体趋势明显向上。

**影响：** 上涨趋势被降级为区间震荡 → 触发"区顶突破"买点 → 实际上不该有二次突破买点。3/13突破后只应等回踩中继，不应再出突破信号。

**建议修复：** 两种互补方案择一或组合使用：

**方案A（宽松极值法）：** 将 `fq = n // 4` 改为 `fq = n // 3`

```python
fq = n // 4  # 旧：3/15=20%
fq = n // 3  # 新：5/15=33%
```

3/16校验：min=5 ≤ fq=5 ✅ → 上涨趋势 ✓

**方案B（突破强制覆盖法，推荐）：** 有效突破（多维评分≥5）后5个交易日内，只要收盘价仍高于突破关键点，结构强制为上涨趋势。不依赖EMA10窗口计算。

```python
# 在 get_structure() 结果后应用
if structure != '上涨趋势':
    if 过去5天内有有效突破(评分≥5) AND 当前收盘价 > 突破关键点:
        return '上涨趋势'
```

**方案B的优点：**
- 直接解决结构问题：3/13突破后3/16不应该退回区间震荡
- 不影响正常结构判定：5天窗口结束后，如果趋势还在，EMA10本身已经跟上来了
- 有兜底机制：如果价格跌破突破关键点，强制覆盖失效，恢复原判定
- 自然解决"上涨趋势中的二次突破"问题——上涨趋势本身就只允许突破和回踩两种买点

**方案A+B组合使用效果最佳：** A解决窗口局部低点带来的普适性问题，B专门解决突破后关键几天的结构稳定问题。

**与"对称末端校验"的区别：** 对称校验解决EMA10数值的滞后问题（末3根已经拐头但EMA10还在高位/低位），中间窗口低点问题是**位置分布**问题（min在窗口中间而非头部）。需分别处理。

详见 `references/2026-05-24-mid-window-dip-misclassification.md`。

## 阶段判断（2026-05-24 已替换为乖离率变化方案）

> **⚠️ 核心改造：中继买点判定已改为乖离率变化+回踩到位三选一，不再依赖 `get_stage()` 返回的阶段。**
> 详见 `references/2026-05-24-乖离率变化替代斜率比阶段判定.md`

### 新方案：乖离率变化三段论（用于中继买点）

旧方法用 `ratio = s2/s1`（EMA10末3根斜率/末15根整体斜率），但这纯粹是数学概念，不是3L量价语言。已替换为乖离率变化方向方案：

```
乖离率(close-EMA10)/EMA10 末3日趋势：
├── >0（扩大）→ 上行/加速
├── <0（收窄）→ 回踩阶段  ← 买点区域
│   + 缩量+小实体 → 供应衰竭 ✅
│   + 地量(<0.6)  → 供应枯竭 ✅ 不限实体
│   + 放量        → 供应出现 ❌
└── ≈0（横盘）→ 整理阶段
    + 缩量 → 缩量整理 ✅
    + 放量 → 放量滞涨 ❌
```

**回踩到位三选一（代码 `_check_pullback`）：**
1. 距关键点支撑 < 2%
2. 乖离率(EMA5/10/20任一)在 -2%~+1%
3. 收盘在EMA5~EMA20之间（均线束内）

**不再依赖stage判定：** 解决了"滞涨/转弱阶段漏判"和"上涨趋势变区间震荡拒买"的问题。

### 旧方法（`get_stage()` 仍保留，used for display only）

`get_stage()` 函数本身未改，仍用EMA10线性回归斜率计算，但**中继买点已不再依赖其返回值**。仅在复盘页面的"阶段"字段展示用。
