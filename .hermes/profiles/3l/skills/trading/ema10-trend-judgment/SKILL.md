# 结构判断 + 阶段判断（3L盯盘版）

## 结构判断

### 算法（终版：EMA10极值 + 双侧末端校验）

**核心逻辑：** 先以EMA10极值位置做初步判定（平滑噪音），再用双侧末端校验修正滞后问题。

**四层判断流程：**

```python
# 第1层：EMA10极值位置（基础判定）
e10 = ema_list(closes, 10)[-15:]       # 全量60天计算EMA10，取末15根
n = len(e10)
max_pos = max(range(n), key=lambda i: e10[i])
min_pos = min(range(n), key=lambda i: e10[i])
first_q = n // 4  # 3
last_q = n - 1 - n // 4  # 11

if max_pos >= last_q and min_pos <= first_q:
    base = '上涨趋势'
elif min_pos >= last_q and max_pos <= first_q:
    base = '下降趋势'
else:
    base = '区间震荡'

l3 = 最近3根EMA10值（去None）

# 第2层：上涨降级（滞后修正）
# 润泽科技案例：EMA10尚在尾部但价格已跌 → 降级
if base == '上涨趋势':
    if l3连续下降 AND close < 最新EMA10:
        return '区间震荡'

# 第3层：下降升级（末端反弹修正）
# 皓元医药案例：EMA10尚在底部但价格已涨 → 升级
if base == '下降趋势':
    if l3连续上升 AND close > 最新EMA10:
        return '区间震荡'

# 第4层：区间震荡→下降（对称校验）
# 天华新能/宁德时代案例：先涨后跌，EMA10最高在中部，最低在头部
# 收盘新低+EMA10下降+20日跌>3%+非巨涨
if base == '区间震荡':
    c15 = closes[-15:]
    if (收盘最低在后3根 AND 收盘最高在前5根
        AND l3连续下降 AND close < 最新EMA10
        AND 20日涨幅 < -3%    # 排除轻微回调
        AND 60日涨幅 < 50%): # 排除暴涨后回调
        return '下降趋势'
```

### ⚠️ 常见坑

1. **EMA10不能从15根K线独立计算** — 必须全量60天计算后取末15根。见下方「算例对比」。
2. **纯收盘价方案太敏感** — 2026-05-20第4轮曾定稿用收盘价但未实现，2026-05-22尝试后被用户否决（华特气体案例：大跌日导致下降趋势假阳性）。
3. **对称校验的20日< -3%阈值** — 2026-05-22讨论中用户指定。太松会把正常回调误判为下降（如药明康德末3天在反弹），太紧会漏掉慢跌。
4. **60日<50%过滤** — 排除暴涨后的正常回调（如应流股份+233%、生益电子+820%）。这些股票EMA10本身可能判下降趋势，但实际是上涨中的回调。

### ⚠️ 对称校验的边界案例（2026-05-22 讨论中待完善）

下降升级条件中的 `20日<-3%` 和 `60日<50%` 是本次讨论中确定的阈值。未来可能遇到更多边界案例需要微调：
- 20日跌幅刚好-2.9%但创新低（如腾远钴业-2.7%被排除）
- 60日涨幅刚好51%但真下降

## 阶段判断

用EMA10线性回归斜率判断，s1=末15根整体斜率（趋势基准），s2=末3根斜率（近期动量，更敏感）。

```python
e10_full = ema_list(closes, 10)        # 全量60天计算EMA10
e10_last = [v for v in e10_full[-15:] if v is not None]  # 取末15根
s1 = _reg_slope(e10_last)              # 末15根整体斜率（趋势基准）
s2 = _reg_slope(e10_last[-3:])         # 末3根斜率（近期动量，更敏感）
```

**⚠️ 坑：阶段判断必须用与结构判断相同的窗口（末15根），不能用全部EMA10。**

### 区间震荡 — 用关键点支撑替代15日极值算位置

**背景：** 区间震荡的阶段判定原用最近15日最高/最低价算分位。但15日内有**单根异常大阳线或长上影**时，会拉升区间顶，导致实际在区间中部的价格被误判为顶部。

**修复方案：** `support_level`/`resistance_level` 由调用方从K线"突"关键点计算后传入，替代原始15日极值。

```python
lo = support_level if support_level else min(lows[-15:])
hi = resistance_level if resistance_level else max(highs[-15:])
pct = (cur - lo) / (hi - lo) * 100
if pct < 30:   → 区间底部
elif pct > 70: → 区间顶部
else:          → 区间中段
```

### 上涨趋势 — 加速/滞涨/上行/转弱

```
s1>0, s2>0 (整体趋势向上，近期也在向上)
  ├─ ratio > 1.8 → 🚀 加速（近期斜率远超整体，快速拉升）
  ├─ ratio < 0.4 → 需结合量能区分：
  │   ├─ 量缩80%+价在EMA10上 → 🔄 缩量整理（中继蓄力，可持股/加仓）
  │   └─ 其他                 → ⚠️ 滞涨（警惕出货，减仓/离场）
  └─ 其他        → ↑ 上行（正常健康趋势）
s1>0, s2<0       → 📉 转弱（整体向上但近期已拐头）

**⚠️ 缩量滞涨 vs 中继买点区分（参考 `2026-05-21-shrink-stagnation-vs-relay.md`）：**
同样的缩量（如量比74%），在ratio≥0.4时是「中继买点」，在ratio<0.4时是「缩量滞涨」。
核心区别不是缩量本身，而是EMA10斜率——斜率正常时缩量=供应被吸收，斜率走平时缩量=无人买。
```

**⚠️ V反误判加速（C方案 2026-05-21）：** 当ratio > 1.8时，检查近6根收盘价是否有单日跌幅>3%的急跌。如有，将s1起点从15根改为急跌后重新算，避免急跌后的V型反弹被误判为加速。详见 `references/2026-05-21-v-reversal-acceleration-fix.md`。

### 下降趋势 — 加速跌/下行/转强

```
s1<0, s2<0 (整体趋势向下，近期也在向下)
  ├─ ratio > 1.8 → 📉 加速跌（近期跌速远超整体）
  └─ 其他        → ↓ 下行（正常下跌）
s1<0, s2>0       → 📈 转强（整体向下但近期已拐头向上）
```

### D方案：判断整理后突破（2026-05-22 终版）

判加速(ratio>1.8)前，检查最近有无确认过程。**有确认过程→整理后突破，降为上行；无确认过程→留加速。** 详见 `references/2026-05-22-d-scheme-final-confirmation-process.md`。
