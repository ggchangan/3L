# get_stage() 算法演进（2026-05-21）

## 版本时间线

### v0: 60根对半切分（初始）
```
s1 = slope(e10[:30])  # 前30根
s2 = slope(e10[30:])  # 后30根
```
**问题：** 前30根含3月老数据，拉升/回调混在一起线性回归趋近于零。润泽科技前30根含涨→跌→涨三段，s1=-0.051被误判为转强。

### v1: 末15根对半切分
```
e10_last = e10[-15:]
s1 = slope(e10_last[:7])
s2 = slope(e10_last[7:])
```
**问题：** 固定切分边界敏感——如果拐点恰好在7/8边界附近，结果不稳定。

### v2: s1=15整体, s2=末5根（最终定稿第一步）
```
s1 = slope(e10[-15:])    # 15根整体趋势基准
s2 = slope(e10[-5:])     # 末5根近期动量
```
**问题：** 用户反馈「最新的是否更应该被体现」，且大族数控的240大阳线拉升区间顶通过支撑位规避了，但拓普集团的加速判定边界(1.84)刚好过线。

### v3: s1=15整体, s2=末3根（当前最终版）
```
s1 = slope(e10[-15:])    # 15根整体趋势基准
s2 = slope(e10[-3:])     # 末3根近期动量（更敏感）
```

**效果对比：**

| 股票 | v1(半切) | v2(末5根) | v3(末3根) | 实际走势 |
|------|---------|----------|----------|---------|
| 润泽科技 | 转强 → ❌ | 上行 → ✅ | 转弱 → ⚠️ | 最近3天回调(99→96)，末3根EMA97.65→97.77→97.42微跌 |
| 拓普集团 | 加速 → ❌ | 上行 → ✅ | 上行 → ✅ | 正常上行，近3天未加速 |
| 伟创电气 | 上行 → ✅ | 加速 → ❌ | 加速 → ❌ | 4天从68→81大涨18.5% |
| 宏发股份 | 转强 → ✅ | 上行 → ✅ | 上行 → ✅ | 稳定上行 |
| 国际复材 | 上行 → ✅ | 滞涨 → ⚠️ | 滞涨 → ⚠️ | 末3根EMA10几乎走平 |

## 关键教训

1. **窗口统一：** 阶段判断必须用末15根EMA10（与结构判断一致），全量60根会引入老数据污染
2. **不等长更合理：** 整体趋势用长窗口(15根)，近期动量用短窗口(3根)，比固定对半切分更合理
3. **s2窗口越短越敏感：** 末3根比末5根更能捕捉到最近的转向信号
4. **EMA10滞后：** 加速判定比K线形态慢1-2天，这是EMA固有特性，不是bug
