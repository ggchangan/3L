# 阶段判定算法演化（2026-05-21）

## 问题链路

润泽科技(300442)在上涨趋势中误判为"转强" → 追查发现阶段判定用了**全部60根EMA10**算斜率 → 60天里的老涨跌（3月从93→83的回调）污染了s1 → 改用末15根后正确。

## 迭代过程

| 版本 | s1 | s2 | 问题 |
|------|----|----|------|
| v1（原版） | 60根EMA10对半切 | 60根EMA10对半切 | 老数据污染，润泽科技误判转强 |
| v2 | 末15根前7根 | 末15根后8根 | 对半切边界敏感，s1/s2各7~8根稀释了近期动量 |
| v3 | 末15根整体 | 末5根 | 用户说5根仍不够敏感（拓普ratio=1.84刚过线） |
| v4 ✅ | 末15根整体 | **末3根** | 用户确认，更灵敏捕捉加速/转弱 |

## 最终定稿

```python
e10_full = ema_list(closes, 10)        # 全量60天计算EMA10
e10_last = [v for v in e10_full[-15:] if v is not None]  # 末15根
s1 = _reg_slope(e10_last)              # 趋势基准
s2 = _reg_slope(e10_last[-3:])         # 近期动量（更敏感）
```

## 关键教训

1. 阶段判定窗口必须与结构判定一致（末15根），s2用更短的窗口（末3根）体现近期动量
2. 不等长窗口比对半切分更稳定——避免拐点恰好在切分边界上的误判
3. 加速/滞涨/转弱/转强的区分完全是本次新增的功能，原版只有上行/下行/转折
