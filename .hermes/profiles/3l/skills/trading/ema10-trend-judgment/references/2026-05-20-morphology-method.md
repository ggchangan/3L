# 2026-05-20: 形态法替代端点对比法 + 线性回归检测加速/滞涨

## 背景

原结构算法用 `e10[-1] vs e10[0]`（15日跨度端点对比），用户质疑电池板块"先涨后跌"却被判为上涨趋势。

## 调试过程

1. 打印全部15天EMA10数据 → 发现EMA10先涨8天(22811→23709)后跌5天(23709→23367)
2. 端点对比法：+1.84% → 判上涨趋势 ❌ 未抓到"先涨后跌"的转折
3. 形态法：9涨7跌(56%) → 区间震荡 ✅

## 新方案

### 结构判断（形态法）

用涨跌天数比例代替端点对比：
```python
up_ratio = ↑天数 / (↑天数 + ↓天数)
if up_ratio >= 0.7: 上涨趋势
elif up_ratio <= 0.3: 下降趋势
else: 区间震荡
```

### 加速/滞涨判断（线性回归半段对比）

用户指定阈值：
- 后半段斜率 / 前半段斜率 > **1.8** → 🚀 加速
- 后半段斜率 / 前半段斜率 < **0.4** → ⚠️ 滞涨

### 区间震荡的阶段

用户指出应是"区间底部/区间顶部/区间中段"（按价格在15日高低点的位置），而非线性回归方向判断。待确认具体实现。

## 缓存陷阱（重复踩坑）

`server.py` 中 `/api/monitor/sectors` 端点原来缓存了整个结构/阶段结果到 `sectors_{date}.json`。代码改对了但缓存不更新，直到删除缓存文件才生效。

**教训**：结构/阶段是计算产物，不应缓存。只有原始K线数据可以缓存。已去掉该缓存层。
