# 结构判断算法进化史（2026-05-20）

## 背景

本轮修正核心原因是：**我自作主张编造了3L中没有的算法**，用户多次纠正才走到正确的收盘价极值位置法。

## 四轮修正

### 第1轮：伪阈值（涨天数比法）

**错误**：用了70%/30%涨天数比阈值判断结构  `up=10/14=71.4% → 上涨趋势`

**用户纠正**：  "不是这样的算法呀 那有什么阈值？" "逻辑是MA10数据 先涨后跌就是区间震荡"

**问题本质**：自己编了3L中没有的固定百分比阈值。

### 第2轮：方向切换计数

**错误**：统计EMA10方向切换次数，白酒13↓1↑被算为2次切换→区间震荡

**用户纠正**："白酒是怎么计算成区间震荡的？"

**问题本质**：单个孤立反向日只是噪声，整体趋势是下行（创了新低）。

### 第3轮：极值位置法（用EMA10）

**算法**：找EMA10最高/最低点在窗口中的位置，四分位法判定

**用户纠正**：光伏设备先涨后跌，但EMA10滞后导致误判为上涨趋势

**问题本质**：EMA10是滞后指标。收盘价在#10见顶后下跌，EMA10到#14才追上峰值。

### 第4轮（定稿）：极值位置法（用收盘价）

**最终算法**：直接用最近15日**收盘价**找最高/最低点位置，四分位判定。

```
前25%        后25%
[0  1  2  3]  ...  [11  12  13  14]

max在尾部 + min在头部 → 上涨趋势
min在尾部 + max在头部 → 下降趋势
极值在中间 → 区间震荡
```

**分界线**：
- 结构用**收盘价**（非滞后，反映真实价格位置）
- 阶段用**EMA10**（反映动能速度变化）

### 关键案例

| 板块 | 收盘价极值 | 结果 | 最终判定 |
|:----|:----------|:-----|:--------|
| 半导体 | min[#0]=14397, max[#14]=17966 | 尾高头低 | 📈上涨趋势 |
| 电池 | min[#0]=22669, max[#5]=24398 | 最高在中间 | ➡区间震荡 |
| 白酒 | max[#0]=2322, min[#13]=2174 | 头高尾低 | 📉下降趋势 |
| 光伏设备 | min[#2]=9059, max[#10]=10041 | 最高在中间 | ➡区间震荡 |

### 教训总结

1. 不要自己编3L中没有的百分比阈值（70%/30%）
2. 不要用EMA10找极值位置（滞后）
3. 结构和阶段用不同数据源：收盘价→结构，EMA10→阶段
4. "看MA10"指的是**用MA10来看价格的方向和位置**，不是用MA10的值本身来计算
