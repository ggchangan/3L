# 关键点法确定支撑/压力（2026-05-20定稿）

## 背景

板块区间震荡时，判断价格在区间内的位置需要确定支撑线和压力线。
使用价格极值（min/max）不准确——60日前的极值不代表当前的有效支撑/压力。

## 定稿方案

**压力线** = 最近的**前高**（第1类关键点，label='前高'，位于当前价格上方）
**支撑线** = 最近的**突破点**（第2类关键点，label='突'，位于当前价格下方）

这是3L体系的逻辑：突破发生后，原压力位转化为支撑位。

## 数据来源

复用 `/api/sector-chart` 端点生成的 `review_charts/sector_{name}_kline.json`：
- 60日OHLCV数据
- 关键点列表（含前高、前低、天量、突破、反转等）

## 备用方案

当某个方向没有对应关键点时，用**最近20日**的价格极值作为备用：
```python
n20 = min(20, len(c60))
hi_fallback = max(h60[-n20:]) if len(h60) >= n20 else max(h60)
lo_fallback = min(l60[-n20:]) if len(l60) >= n20 else min(l60)
```

## 案例：电池

- 60日Kline中最近的前高（上方）：23337（索引40，前高）
- 60日Kline中最近的突破点（下方）：22896（索引39，突）
- 当前价：23115
- 区间位置：(23115-22896)/(23337-22896) = 49.6% → 区间中段 ✅

对比旧方案：
- 60日min/max：74.9%(用19008为底) → ❌ 区间顶部
- 20日min/max：40.3%(用22187为底) → 区间中段（但无关键点支撑）

## 实现位置

- `monitor_data.py` → `fill_sector_list()` 函数中的 `else:` 分支（区间震荡处理）
- `server.py` → `/api/sector-chart` 端点中的 `else:` 分支（区间震荡处理）
- 两个位置使用完全相同的算法和阈值

## 关键点发现流程

1. 从 `sector_{name}_kline.json` 读取 `key_points` 列表
2. 筛选 label='突' 且 y<cur 的 → 排序取最接近的（降序第1个）
3. 筛选 label='前高' 且 y>cur 的 → 排序取最接近的（升序第1个）
4. 计算百分位：(cur - lo) / (hi - lo) × 100
5. <30% → 区间底部，>70% → 区间顶部，其他→区间中段
