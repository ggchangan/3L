# V反误判加速修复（C方案）

## 背景

2026-05-21 发现 `ema_utils.py` 的 `get_stage()` 对 **伟创电气(688698)** 误判为加速。

## 问题分析

伟创电气近15日走势：涨→回调→V型反弹。5/14出现单日-3.9%急跌（68.88），随后4天反弹至81.59。

```
s1(15根EMA10整体) = +0.522 ← 包含5/11-5/14的回调段，基线偏低
s2(末3根EMA10)    = +1.249 ← 仅含5/18-5/20的反弹段，斜率较陡
ratio = 1.249/0.522 = 2.39 > 1.8 → 误判为加速 ❌
```

**根因：** s1的15根窗口包含了回调段，让趋势基线偏低；而s2只捕捉到了反弹段。V型反弹不等于趋势加速。

## 修复方案（C方案）

当 `ratio > 1.8`（即将判定加速）时，额外检查近6根收盘价是否有单日跌幅>3%的急跌：

```python
if ratio > 1.8 and len(closes) >= 6:
    check = closes[-6:]
    for ci in range(1, len(check)):
        if (check[ci] - check[ci-1]) / check[ci-1] < -0.03:
            n_after = len(check) - ci   # 急跌后还有多少根
            start = max(1, len(e10_last) - n_after + 1)
            if start < len(e10_last) - 2:
                s1_adj = _reg_slope(e10_last[start:])
                if s1_adj > 0:
                    ratio = s2 / s1_adj  # 用调整后的ratio判加速
            break
```

**伟创电气修复效果：**
```
s1_adj(从急跌5/14后算) = +1.150
调整后ratio = 1.249/1.150 = 1.09 → 上行 ✅
```

## 影响

此修复只影响**同时满足两个条件**的股票：
1. `ratio > 1.8`（即斜率比达到加速阈值）
2. 近5日内有单日跌幅>3%的急跌

正常加速（无急跌背景）不受影响。

## 对比案例：滞涨 vs C方案

**国际复材(301526, 2026-05-21) — 滞涨，C方案不触发：**

```
s1(末15根EMA10斜率) = +0.3157   ← EMA10从14.32涨到18.43
s2(末3根EMA10斜率)  = +0.0778   ← 最后3根EMA10: 18.27→18.35→18.43
ratio = 0.0778/0.3157 = 0.2464

判定：ratio < 0.4 → 滞涨 ✅
```

**国际复材与伟创电气的对比，说明C方案只狭窄地覆盖加速误判场景：**

| 条件 | 伟创电气（加速误判） | 国际复材（滞涨） |
|------|:-------------------:|:---------------:|
| s1>0, s2>0 | ✅ | ✅ |
| ratio>1.8（初始） | ✅ 2.39 | ❌ 0.25 |
| 近5日有>3%急跌 | ✅ 5/14 -3.9% | ❌ |
| C方案触发 | ✅ | ❌（ratio<1.8无需检查） |
| 最终阶段 | 上行（1.09） | 滞涨（0.25） |

**验证价值：** 如果一个股票ratio<0.4走滞涨分支，说明EMA10斜率在急速放缓（s2<<s1），与加速反向——这说明C方案检测仅在ratio>1.8的门槛内生效，不会误闯入滞涨/正常上行区间。
