# 缩量整理 vs 滞涨 — 量能区分逻辑（2026-05-21）

## 背景

`get_stage()` 中当 `ratio = s2/s1 < 0.4`（涨不动）时，之前统一判为"滞涨"（卖出/警惕信号）。
但**涨不动**可能是两种完全不同的场景：

1. **滞涨（危险）** — 放量出货，价格在EMA10附近反复
2. **缩量整理（中继蓄力）** — 卖压枯竭，价格仍在EMA10上方

## 算法

```python
# get_stage() 中 ratio < 0.4 分支
if ratio < 0.4:
    if volumes and len(volumes) >= 13 and closes[-1] > e10_last[-1]:
        vol_last3 = sum(volumes[-3:]) / 3
        vol_prev10 = sum(volumes[-13:-3]) / 10
        if vol_prev10 > 0 and vol_last3 / vol_prev10 < 0.8:
            return '缩量整理'
    return '滞涨'
```

**两个条件必须同时满足：**
- 量比（末3日均量/前10日均量）< 80%
- 最新收盘价 > 最新EMA10

前端stageColors映射：
- 缩量整理 → `#ffd700` 金色（中性等待，非卖出）
- 滞涨 → `#ff6b6b` 橙红（警惕）

## 实例：国际复材（301526）

```
ratio = 0.2464 < 0.4 → 涨不动，进入量能分支

末3日均量: 1,393,139
前10日均量: 2,156,469
量比: 65% < 80% → 缩量 ✅
最新收盘: 18.78 > EMA10: 18.43 → 价在EMA10上 ✅

结论: 缩量整理
```

K线验证：
- 5/13 +14.48% 涨停式突破，量2,809,642
- 后连续5天缩量：1,856,959 → 2,187,025 → 1,520,501 → 1,273,301 → 1,385,614
- 卖压明显枯竭，非出货

## 操作含义

| 阶段 | 含义 | 操作建议 |
|------|------|---------|
| 缩量整理 | 中继蓄力，卖压枯竭 | 持有，可加仓 |
| 滞涨 | 资金出货，卖压持续 | 警惕，考虑减仓/离场 |

## 调用方需注意

`generate_review_data.py` 的 `get_buy_sell_signals()` 从 `all_stocks_60d.json` 读vol数据。

**踩坑：** `all_stocks_60d.json` 用 `volume` 字段名（不是 `vol`）。必须在调用处做双字段兼容：
```python
vols_60 = [k.get('volume', k.get('vol', 0)) for k in _kls]
```
否则 vols_60 全为0，`vol_prev10 > 0` 不满足，永远判"滞涨"。
